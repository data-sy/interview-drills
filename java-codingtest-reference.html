입출력 · 정렬

01입출력 (표준 템플릿)

Scanner는 편하지만 느리다. 입력이 많으면 BufferedReader + StringBuilder가 정석.

빠른 입출력 템플릿 (권장)

Scanner (간단한 문제용)

실전 팁
  • 입력이 수만 줄 이상이면 Scanner는 시간초과 위험 → BufferedReader.
  • 출력이 많으면 System.out.println 반복 대신 StringBuilder에 모아 마지막에 한 번 출력.
  • br.readLine()은 IOException을 던지므로 main(...) throws IOException.

02정렬 (기준 커스텀)

코테 절반은 "정렬하고 나면 쉬워지는" 문제. 다중 기준 정렬을 자유롭게.

실전 팁
  • 기본형 배열(int[])은 내림차순 정렬 API가 없다 → Integer[]로 바꾸거나 오름차순 후 뒤집기.
  • 비교식 x - y는 값이 크면 오버플로 위험 → 안전하게 Integer.compare(x, y).
완전탐색 · 재귀

03완전탐색 (브루트포스)

N이 작을 때 모든 경우를 다 확인. 중첩 반복문·비트마스크가 대표적.

실전 팁
  • 1 << n = 2ⁿ. mask & (1 << i)로 i번째 원소 선택 여부 확인.
  • N ≤ 20이면 2²⁰ ≈ 100만 → 비트마스크 완전탐색 가능.

04재귀 · 백트래킹

조합·순열을 만들며 탐색하고, 아니면 되돌린다(선택 → 재귀 → 선택 취소).

순열 (nPr) / 조합 (nCr)

백트래킹 (가지치기) — N-Queen 골격

실전 팁
  • 백트래킹의 핵심: 선택 → 재귀 → 선택 취소. 상태를 공유(필드/배열)하면 반드시 원복.
  • 불가능한 경로를 조기에 잘라내는 가지치기(isSafe 등)가 성능을 결정한다.
  • 종료 조건(base case)을 먼저 쓰고, 무한재귀·인덱스 범위를 항상 확인.
그래프 탐색

05BFS (너비 우선 · 최단거리)

큐로 가까운 곳부터. 가중치 없는 그래프의 최단 거리는 BFS가 정답.

격자(2D) BFS — 미로 최단거리

인접 리스트 그래프 BFS

실전 팁
  • visited는 큐에 넣는 시점에 표시한다(꺼낼 때 하면 중복 삽입으로 폭발).
  • 가중치가 모두 1이면 BFS = 최단거리. 가중치가 다르면 다익스트라(우선순위 큐).
  • 격자 이동은 dx/dy 배열로 4방향(또는 8방향)을 반복문 하나로 처리.

06DFS (깊이 우선)

재귀(또는 스택)로 끝까지 파고든다. 연결요소·경로 탐색·백트래킹의 기반.

실전 팁
  • 재귀 DFS는 깊이가 크면(수십만) StackOverflow 위험 → 그럴 땐 스택 자료구조로 반복 구현.
  • BFS vs DFS: 최단거리는 BFS, 모든 경로·조합 탐색·연결요소는 DFS가 자연스럽다.
탐색 · 구간 · DP

07이분탐색 (Binary Search)

정렬된 데이터에서 O(log N). "~이하 최대/이상 최소" lower/upper bound, 파라메트릭 서치가 핵심.

실전 팁
  • 이분탐색 전제: 데이터가 정렬되어 있어야 한다.
  • "최댓값을 최소로 / 최소 시간" 같은 최적화 문제는 답을 mid로 가정하고 가능한지(possible) 판정하는 파라메트릭 서치로 푼다.
  • (lo+hi)/2는 오버플로 위험 → (lo+hi)>>>1 또는 lo+(hi-lo)/2.
  • Java 내장: Arrays.binarySearch(a, key), Collections.binarySearch(list, key).

08투포인터 · 슬라이딩 윈도우

두 포인터로 구간을 좁히거나 넓힌다. 정렬 배열의 쌍 찾기, 연속 부분합에 강력.

실전 팁
  • 양 끝 투포인터는 보통 정렬이 전제. 연속 구간(슬라이딩 윈도우)은 정렬 없이 원본 순서로.
  • O(N²) 이중 반복이 시간초과일 때 투포인터로 O(N)으로 줄이는 게 전형적 패턴.

09누적합 (Prefix Sum)

구간 합을 여러 번 물어볼 때, 미리 누적합을 만들어 각 질의를 O(1)로.

실전 팁
  • "구간 합을 Q번 질의" → 매번 더하면 O(NQ), 누적합이면 전처리 O(N) + 질의 O(1).
  • 인덱스 오프바이원(off-by-one) 실수 방지로 1-based 누적합 배열(크기 N+1)을 쓰는 걸 권장.

10DP (동적 계획법)

큰 문제를 작은 부분문제로. 점화식을 세우고 메모이제이션(하향식) 또는 테이블(상향식).

1차원 DP — 피보나치 / 계단 오르기

2차원 DP — 0/1 배낭(Knapsack)

실전 팁
  • DP 3요소: ① 상태 정의(dp[i]가 뭘 의미하는지) ② 점화식 ③ 초기값.
  • 완전탐색에 "겹치는 부분문제"가 보이면 메모이제이션으로 DP 전환.
  • 메모 초기값이 0과 겹치면(fib(0)=0) Arrays.fill(memo, -1)로 "미계산" 구분.
