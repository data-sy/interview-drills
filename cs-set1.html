Q1객관식 · 배열

배열(Array)의 특성으로 옳은 것은?

  1. 인덱스로 임의 원소에 O(1)로 접근할 수 있다
  2. 중간에 원소를 삽입해도 다른 원소 이동이 없다
  3. 크기가 런타임에 자유롭게 늘어난다(정적 배열 기준)
  4. 원소들이 메모리에 흩어져 저장된다
정답 확인
① 인덱스로 임의 원소에 O(1)로 접근할 수 있다
배열은 연속된 메모리에 원소를 저장하므로 시작 주소+인덱스×크기로 즉시 접근(O(1))한다. 중간 삽입/삭제는 뒤 원소를 밀어야 해 O(n)이다. SI에서 조회 위주의 코드 테이블(공통코드)을 배열/리스트로 캐싱할 때 이 임의접근 이점이 성능의 근거가 된다.
Q2객관식 · Big-O 표기

시간복잡도 O(1), O(log n), O(n), O(n²)를 빠른(작은) 순서대로 나열한 것은?

  1. O(n) < O(1) < O(log n) < O(n²)
  2. O(1) < O(n) < O(log n) < O(n²)
  3. O(log n) < O(1) < O(n²) < O(n)
  4. O(1) < O(log n) < O(n) < O(n²)
정답 확인
④ O(1) < O(log n) < O(n) < O(n²)
입력 n이 커질수록 증가율이 상수 < 로그 < 선형 < 제곱 순이다. Big-O는 최악의 경우 증가 추세를 나타낸다. SI에서 대량 통계 데이터를 다룰 때 O(n²) 이중 루프가 데이터 폭증 시 응답 지연의 원인이 되므로 설계 단계에서 복잡도를 가늠한다.
Q3객관식 · 프로세스 vs 스레드

프로세스와 스레드의 차이로 옳은 것은?

  1. 스레드는 각자 독립된 코드/데이터/힙 영역을 완전히 분리해 가진다
  2. 프로세스 간에는 기본적으로 메모리를 공유하지 않는다
  3. 스레드 생성 비용이 프로세스 생성 비용보다 항상 크다
  4. 한 프로세스는 반드시 하나의 스레드만 가질 수 있다
정답 확인
② 프로세스 간에는 기본적으로 메모리를 공유하지 않는다
프로세스는 독립된 메모리 공간을 가져 서로 격리되고, 스레드는 같은 프로세스 안에서 코드/데이터/힙을 공유하고 스택만 따로 가진다. 스레드 생성이 더 가볍다. SI 배치 프로그램에서 다중 스레드로 파일 병렬 처리를 할 때 공유 자원(공유 변수) 동기화가 이슈가 된다.
Q4객관식 · OSI 7계층

OSI 7계층 중 IP 주소를 기반으로 경로(라우팅)를 결정하는 계층은?

  1. 데이터링크 계층(2계층)
  2. 전송 계층(4계층)
  3. 네트워크 계층(3계층)
  4. 응용 계층(7계층)
정답 확인
③ 네트워크 계층(3계층)
네트워크 계층(3계층)은 IP 주소로 종단 간 경로를 정하는 라우팅을 담당한다. 데이터링크는 MAC 주소 기반 인접 노드 전달, 전송은 포트 기반 프로세스 간 전달이다. 고객사 방화벽/네트워크 이슈를 진단할 때 어느 계층 문제인지 구분하는 것이 장애 대응의 출발점이다.
Q5객관식 · DB 정규화

데이터베이스 정규화의 주된 목적으로 가장 옳은 것은?

  1. 데이터 중복을 줄여 이상현상을 방지한다
  2. 테이블 수를 최소화해 조인을 없앤다
  3. 인덱스를 자동으로 생성한다
  4. 항상 조회 속도를 최우선으로 높인다
정답 확인
① 데이터 중복을 줄여 이상현상을 방지한다
정규화는 함수 종속을 기준으로 테이블을 분해해 중복을 제거하고 삽입/삭제/갱신 이상현상을 막는 것이 목적이다. 오히려 조인이 늘어 조회는 느려질 수 있어 실무에선 반정규화로 절충한다. 통계 시스템에서 집계 조회 성능을 위해 의도적 반정규화 설계를 하는 이유가 여기 있다.
Q6객관식 · 스택

스택(Stack)에 순서대로 1, 2, 3을 push한 뒤 pop을 두 번 하면 나오는 값은?

  1. 1, 2
  2. 3, 2
  3. 2, 1
  4. 1, 3
정답 확인
② 3, 2
스택은 LIFO(후입선출)라서 마지막에 넣은 3이 먼저, 그다음 2가 나온다. 함수 호출 스택, 실행 취소(undo), 괄호 검사 등이 대표 응용이다. SI 웹 화면의 "이전으로" 뒤로가기 이력 관리도 스택 구조로 구현하는 경우가 많다.
Q7객관식 · 진법 변환

2진수 1011(2)를 10진수로 변환한 값은?

  1. 9
  2. 10
  3. 11
  4. 13
정답 확인
③ 11
1×2³ + 0×2² + 1×2¹ + 1×2⁰ = 8+0+2+1 = 11이다. 각 자리의 가중치를 더하는 위치 표기법이다. SI에서 권한 플래그를 비트로 관리(예: 읽기1·쓰기2·실행4)할 때 진법 감각이 값 해석의 기초가 된다.
Q8객관식 · 이진 탐색

이진 탐색(Binary Search)을 사용하기 위한 필수 전제 조건은?

  1. 데이터 개수가 2의 거듭제곱이어야 한다
  2. 데이터가 연결리스트로 저장되어야 한다
  3. 데이터가 모두 정수여야 한다
  4. 데이터가 정렬되어 있어야 한다
정답 확인
④ 데이터가 정렬되어 있어야 한다
이진 탐색은 중간값과 비교해 탐색 범위를 절반씩 줄이므로 반드시 정렬된 데이터가 필요하며 O(log n)이다. 정렬 안 된 데이터는 선형 탐색(O(n))을 써야 한다. 고객사 대용량 코드 목록에서 정렬 후 이진 탐색을 쓰면 조회 성능이 크게 좋아진다.
Q9단답 · 스택/큐

먼저 들어온 데이터가 먼저 나가는(FIFO) 자료구조의 이름과, 나중에 들어온 데이터가 먼저 나가는(LIFO) 자료구조의 이름을 각각 쓰시오.

모범답안 확인
모범답안
FIFO = 큐(Queue), LIFO = 스택(Stack)

큐는 한쪽(rear)에서 넣고 반대쪽(front)에서 빼는 선입선출, 스택은 한쪽 끝에서만 넣고 빼는 후입선출이다. SI 대량 작업 처리에서 요청을 순서대로 처리하는 작업 대기열은 큐, 실행 취소 이력은 스택으로 구현한다.
Q10서술형 · 교착상태

교착상태(Deadlock)가 발생하기 위한 4가지 필요조건을 모두 쓰고, 각각을 한 줄로 설명하시오.

모범답안 확인
모범답안
① 상호배제(Mutual Exclusion): 자원을 한 번에 하나의 프로세스만 사용. ② 점유와 대기(Hold and Wait): 자원을 점유한 채 다른 자원을 기다림. ③ 비선점(No Preemption): 다른 프로세스의 자원을 강제로 뺏을 수 없음. ④ 환형 대기(Circular Wait): 프로세스들이 원형으로 서로의 자원을 대기.

이 네 조건이 동시에 성립할 때만 교착상태가 생기므로, 하나라도 깨면 예방된다(채점 포인트: 4개 명칭+의미). SI 배치/트랜잭션에서 여러 테이블을 서로 다른 순서로 락 걸 때 환형 대기가 생겨 DB 데드락이 나므로, 락 획득 순서를 통일하는 것이 실무 해법이다.

이 세트 우선복습 3줄

  1. Big-O 증가율 순서 — O(1) < O(log n) < O(n) < O(n²). 상수·로그·선형·제곱 순으로 커지며, 이중 루프 O(n²)가 데이터 폭증 시 병목이 된다.
  2. 프로세스 vs 스레드 — 프로세스는 메모리를 격리(공유 X), 스레드는 코드/데이터/힙을 공유하고 스택만 따로. 스레드 생성이 더 가볍다.
  3. 이진 탐색의 전제 — 반드시 정렬된 데이터에서만 O(log n)으로 동작. 정렬 안 됐으면 선형 탐색(O(n))이 불가피하다.
