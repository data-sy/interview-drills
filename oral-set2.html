다형성·캡슐화부터 정규화·TCP/UDP·형상관리까지

질문을 보고 소리 내어 말하듯 답안을 적은 뒤 모범답변 확인을 눌러 요지와 비교하고, 스스로 O/X로 채점하세요.

Q11구두 · 다형성

다형성이 무엇이고 실제로 어떤 이점이 있는지 설명해보세요.

모범답변 요지
  • 같은 타입(부모/인터페이스) 참조로 여러 구현 객체를 다룰 수 있음
  • 오버라이딩과 업캐스팅을 통해 실행 시점에 실제 객체의 메서드가 호출됨(동적 바인딩)
  • 새 구현을 추가해도 호출 코드는 바꾸지 않아도 됨(확장에 열림)
  • 실무 연결: 결제 수단·연동 대상이 늘어나는 SI 프로젝트에서 인터페이스 기반 다형성으로 분기문 없이 확장 가능하다는 점 언급
Q12구두 · 캡슐화

필드를 private으로 숨기고 getter/setter로 접근하게 하는 이유가 무엇인가요?

모범답변 요지
  • 내부 데이터를 직접 노출하지 않아 잘못된 값 대입을 막음(유효성 검증 삽입 가능)
  • 내부 구현을 바꿔도 외부 사용 코드에 영향이 없음(정보 은닉)
  • 읽기 전용·쓰기 전용 등 접근 범위를 제어 가능
  • 실무 연결: 여러 사람이 오래 손대는 SI 코드에서 값이 아무 데서나 바뀌면 버그 추적이 어려우므로, 접근 통로를 하나로 모으는 것이 유지보수에 유리
Q13구두 · 고객사 요구 변경

개발이 절반쯤 진행됐는데 고객이 요구사항을 바꾸자고 합니다. 어떻게 대응하겠습니까?

모범답변 요지
  • 감정적으로 거절하지 않고 변경의 배경·목적을 먼저 파악
  • 변경이 일정·범위·다른 기능에 미치는 영향을 분석해 공유
  • 우선순위를 협의하고, 합의된 변경은 문서·이슈로 기록
  • 팀 리더·PM에 보고해 혼자 결정하지 않음
  • 실무 연결: SI는 요구 변경이 상시 발생하므로, 변경 자체를 막기보다 영향 분석과 기록으로 관리하는 태도가 중요함을 강조
Q14구두 · 정규화는 왜

데이터베이스 정규화를 하는 이유와, 무조건 좋은지에 대해 말해보세요.

모범답변 요지
  • 중복 데이터를 제거해 삽입·수정·삭제 이상(anomaly)을 방지
  • 데이터 일관성과 무결성을 높임
  • 다만 조인이 늘어 조회 성능이 떨어질 수 있어, 통계·조회 위주 시스템은 의도적 반정규화(denormalization)도 함
  • 실무 연결: 통계·리포트 중심 SI 시스템에서는 성능을 위해 일부 반정규화를 택하는 트레이드오프 판단이 실무 역량임을 언급
Q15구두 · 인덱스 원리

인덱스를 걸면 왜 조회가 빨라지고, 어떤 비용이 따르는지 설명해보세요.

모범답변 요지
  • 정렬된 자료구조(B-tree)로 전체 스캔 대신 원하는 위치를 빠르게 탐색
  • 대신 INSERT/UPDATE/DELETE 시 인덱스도 갱신해야 해 쓰기 성능·저장공간 비용 발생
  • 카디널리티 높은 컬럼·WHERE·JOIN·ORDER BY에 자주 쓰이는 컬럼이 후보
  • 실무 연결: 데이터가 쌓인 고객사 운영 DB에서 느린 조회의 1차 진단이 인덱스 유무이며, 남발하면 배치 성능이 나빠진다는 균형 감각 강조
Q16구두 · SQL 튜닝

특정 화면 조회가 갑자기 느려졌다면, 원인을 어떻게 찾고 개선하겠습니까?

모범답변 요지
  • 실행계획(EXPLAIN)을 확인해 풀스캔·비효율 조인 여부 점검
  • WHERE·JOIN 조건 컬럼의 인덱스 유무와 함수 사용으로 인한 인덱스 미사용 확인
  • 불필요한 컬럼·서브쿼리 정리, SELECT * 지양
  • 데이터 증가·통계정보 갱신 여부도 확인
  • 실무 연결: 운영 데이터가 누적되면서 느려지는 것이 SI 흔한 이슈라, 실행계획을 읽는 습관이 튜닝의 출발점임을 강조
Q17구두 · MVC란

MVC 패턴이 무엇이고 왜 역할을 나누는지 설명해보세요.

모범답변 요지
  • Model(데이터·비즈니스 로직), View(화면 표현), Controller(요청 처리·흐름 제어)로 분리
  • 관심사 분리로 화면과 로직을 독립적으로 수정 가능
  • 재사용성·테스트 용이성·협업 분담이 쉬워짐
  • 실무 연결: 화면 담당과 로직 담당이 나뉘는 SI 팀 개발에서 MVC 분리가 병렬 작업과 유지보수를 가능하게 함을 언급
Q18구두 · TCP vs UDP

TCP와 UDP의 차이와 각각 어디에 쓰는지 말해보세요.

모범답변 요지
  • TCP: 연결 지향(3-way handshake), 순서 보장·재전송으로 신뢰성 높음
  • UDP: 비연결, 빠르지만 신뢰성 보장 없음
  • TCP는 웹·파일 전송·결제 등 정확성 중요한 곳, UDP는 스트리밍·실시간 등 속도 우선
  • 실무 연결: 대부분의 SI 업무 시스템(HTTP·DB 연동)이 TCP 기반이라 신뢰성 보장 메커니즘을 이해하는 것이 기본임을 언급
Q19구두 · 형상관리(Git)

팀에서 Git으로 협업할 때 브랜치와 커밋을 어떻게 관리하는 게 좋다고 생각하나요?

모범답변 요지
  • main(운영)은 안정 버전 유지, 기능별로 브랜치를 나눠 작업 후 병합
  • 커밋은 의미 단위로 작게, 메시지에 무엇을·왜를 명확히
  • 병합 전 코드리뷰·충돌 해결, 함부로 force push 지양
  • 실무 연결: 여러 명이 같은 소스를 다루는 SI 프로젝트에서 형상관리 규칙이 무너지면 배포 사고로 이어지므로 브랜치 전략과 커밋 규율을 강조
Q20구두 · 강점

본인의 강점은 무엇이고 그것을 보여준 경험이 있나요?

모범답변 요지
  • 구체적 강점 하나를 선택(예: 끈기 있는 문제 추적력, 꼼꼼한 기록 습관)
  • 그 강점을 발휘한 실제 경험(프로젝트·과제)을 근거로 제시
  • 그 강점이 SI 업무(유지보수·협업)에 어떻게 도움 되는지 연결
  • 실무 연결: 추상적 자기 칭찬이 아니라, SI에서 필요한 성실함·소통·학습력과 연결된 강점을 사례와 함께 말하는 것이 핵심

이 세트 우선복습 3줄

  1. 정규화는 만능이 아니다 — 중복 제거로 이상(anomaly)을 막지만 조인이 늘어 조회가 느려지므로, 통계·리포트 시스템은 의도적 반정규화 트레이드오프를 판단한다.
  2. 인덱스의 양면 — 조회는 B-tree 탐색으로 빨라지지만 INSERT/UPDATE/DELETE마다 갱신 비용·저장공간이 들어, 남발하면 쓰기·배치가 느려진다.
  3. TCP vs UDP — TCP는 연결 지향·순서 보장·재전송(신뢰성), UDP는 비연결·고속(신뢰성 없음). 정확성이 필요하면 TCP, 실시간 속도가 우선이면 UDP.
