Q11객관식 · 연결리스트

배열과 비교했을 때 연결리스트(Linked List)의 장점으로 옳은 것은?

  1. 인덱스로 임의 접근이 O(1)로 빠르다
  2. 삽입/삭제 시 원소 이동 없이 포인터만 바꾸면 된다
  3. 메모리를 연속된 공간에 저장해 캐시 효율이 높다
  4. 저장에 별도의 포인터 공간이 필요 없다
정답 확인
② 삽입/삭제 시 원소 이동 없이 포인터만 바꾸면 된다
연결리스트는 노드가 포인터로 연결되어 삽입/삭제 시 링크만 조정하면 되지만(해당 위치를 알 때 O(1)), 임의 접근은 앞에서부터 따라가야 해 O(n)이다. 잦은 삽입/삭제가 있는 실무 목록(장바구니 등) 구현에서 선택 근거가 된다.
Q12객관식 · 정렬(버블)

버블 정렬(Bubble Sort)의 평균/최악 시간복잡도는?

  1. O(n²)
  2. O(n log n)
  3. O(n)
  4. O(log n)
정답 확인
① O(n²)
버블 정렬은 인접한 두 원소를 비교·교환하며 이중 루프를 돌아 O(n²)다. 구현은 쉽지만 데이터가 많으면 느리다. SI에서 정렬 대상이 수만 건 이상이면 병합·퀵(O(n log n)) 또는 DB의 ORDER BY(인덱스 활용)에 맡기는 것이 실무적으로 낫다.
Q13객관식 · 동기/비동기

동기(Synchronous)와 비동기(Asynchronous)에 대한 설명으로 옳은 것은?

  1. 동기는 요청 후 결과를 기다리지 않고 다음 작업을 진행한다
  2. 비동기는 작업 완료 여부를 호출한 쪽이 책임지고 기다린다
  3. 동기는 요청 결과가 나올 때까지 기다린 뒤 다음을 진행한다
  4. 동기·비동기는 반드시 멀티스레드에서만 구분된다
정답 확인
③ 동기는 요청 결과가 나올 때까지 기다린 뒤 다음을 진행한다
동기는 호출한 작업의 결과를 기다렸다가 반환받아 다음을 진행하고, 비동기는 결과를 기다리지 않고 콜백/이벤트로 나중에 통지받는다. SI 화면에서 대용량 리포트 생성을 비동기로 처리하고 완료 시 알림을 주면 사용자 대기 체감이 크게 준다.
Q14객관식 · TCP vs UDP

TCP와 UDP를 비교한 설명으로 옳은 것은?

  1. TCP는 비연결형, UDP는 연결형이다
  2. TCP는 데이터 순서와 신뢰성을 보장하지만 UDP는 보장하지 않는다
  3. UDP는 3-way handshake로 연결을 수립한다
  4. TCP가 UDP보다 항상 전송 속도가 빠르다
정답 확인
② TCP는 데이터 순서와 신뢰성을 보장하지만 UDP는 보장하지 않는다
TCP는 연결형으로 순서·재전송·흐름제어로 신뢰성을 보장하고, UDP는 비연결형이라 빠르지만 손실·순서를 보장하지 않는다. 파일 전송/결제 같은 정확성 요구엔 TCP, 실시간 스트리밍/DNS엔 UDP를 쓴다. 고객사 연계 방식 결정 시 신뢰성 요구 수준으로 선택한다.
Q15객관식 · 이상현상

정규화가 안 된 테이블에서 발생하는 이상현상(Anomaly)이 아닌 것은?

  1. 삽입 이상(Insertion Anomaly)
  2. 삭제 이상(Deletion Anomaly)
  3. 갱신 이상(Update Anomaly)
  4. 조인 이상(Join Anomaly)
정답 확인
④ 조인 이상(Join Anomaly)
이상현상은 삽입·삭제·갱신 세 가지이며 "조인 이상"이라는 용어는 없다. 중복 데이터 때문에 원치 않는 자료가 함께 들어가거나 사라지거나 일부만 수정되는 문제다. SI에서 고객·주소를 한 테이블에 몰아넣으면 주소만 바꿔도 여러 행을 다 고쳐야 하는 갱신 이상이 생긴다.
Q16객관식 · 큐

큐(Queue)를 활용하기에 가장 적절한 상황은?

  1. 프린터의 인쇄 작업 대기 순서 관리
  2. 함수 호출의 복귀 주소 관리
  3. 수식의 괄호 짝 검사
  4. 웹 브라우저의 뒤로가기 이력 관리
정답 확인
① 프린터의 인쇄 작업 대기 순서 관리
큐는 FIFO라 먼저 요청된 작업을 먼저 처리하는 대기열(프린터 스풀, 작업 큐)에 적합하다. 나머지는 모두 LIFO 특성의 스택 응용이다. SI 시스템에서 몰리는 요청을 순서대로 소화하는 메시지 큐(예: 배치 작업 대기열)가 대표 사례다.
Q17객관식 · 캐시·지역성

캐시(Cache)가 성능을 높이는 근거인 "참조 지역성(Locality)"에 대한 설명으로 옳은 것은?

  1. 최근 접근한 데이터를 다시 접근할 가능성이 높은 것은 공간 지역성이다
  2. 인접한 주소의 데이터를 곧 접근할 가능성이 높은 것은 시간 지역성이다
  3. 시간 지역성과 공간 지역성 덕분에 캐시 적중률이 높아진다
  4. 지역성이 높을수록 캐시 미스(miss)가 늘어난다
정답 확인
③ 시간 지역성과 공간 지역성 덕분에 캐시 적중률이 높아진다
시간 지역성(최근 것 재접근)과 공간 지역성(인접 주소 접근)이 있어 자주/곧 쓸 데이터를 캐시에 두면 적중률이 오른다(보기 1·2는 정의가 뒤바뀜). SI에서 공통코드·설정값을 애플리케이션 캐시에 올려두는 최적화가 이 원리에 기댄다.
Q18객관식 · HTTP 상태코드

클라이언트가 요청한 자원을 찾을 수 없을 때 서버가 반환하는 HTTP 상태코드는?

  1. 200 OK
  2. 301 Moved Permanently
  3. 500 Internal Server Error
  4. 404 Not Found
정답 확인
④ 404 Not Found
404는 요청한 자원이 없음을 뜻하는 4xx(클라이언트 오류)다. 2xx는 성공, 3xx는 리다이렉트, 5xx는 서버 오류다. SI 웹 유지보수 중 로그의 상태코드로 문제 주체(클라이언트 요청 오류인지 서버 장애인지)를 빠르게 가른다.
Q19단답 · 선형 탐색

정렬되지 않은 n개의 데이터에서 특정 값을 찾는 선형 탐색(Linear Search)의 최악 시간복잡도를 Big-O로 쓰시오.

모범답안 확인
모범답안 · O(n)
최악의 경우 맨 끝에 있거나 없어서 n개를 모두 확인해야 하므로 O(n)이다. 정렬이 안 되어 있으면 이진 탐색을 못 써 선형 탐색이 불가피하다. SI에서 정렬 인덱스가 없는 대량 데이터를 매번 전체 스캔하면 느려지므로 인덱스 설계가 중요해진다.
Q20서술형 · 트랜잭션 ACID

트랜잭션의 ACID 4가지 특성의 이름을 모두 쓰고 각각 한 줄로 설명하시오.

모범답안 확인
모범답안
① 원자성(Atomicity): 트랜잭션의 작업은 전부 성공하거나 전부 실패(롤백). ② 일관성(Consistency): 실행 전후 DB가 정합한 상태를 유지. ③ 격리성(Isolation): 동시 실행 트랜잭션이 서로 간섭하지 않음. ④ 지속성(Durability): 커밋된 결과는 장애가 나도 영구 보존.

채점 포인트는 4개 명칭(원자성·일관성·격리성·지속성)과 각 의미다. SI 금융/통계 시스템에서 이체·집계 같은 다단계 작업이 중간에 끊겨도 원자성·지속성 덕에 데이터가 깨지지 않으며, 이것이 DB에 트랜잭션 처리를 맡기는 이유다.

이 세트 우선복습 3줄

  1. 동기 vs 비동기 — 동기는 결과가 나올 때까지 기다린 뒤 다음 진행, 비동기는 안 기다리고 콜백/이벤트로 나중에 통지받는다. "동기가 안 기다린다"는 함정에 주의.
  2. 이상현상은 3종뿐 — 삽입·삭제·갱신 이상만 존재하고 "조인 이상"이라는 용어는 없다.
  3. 참조 지역성 정의 — 시간 지역성 = 최근 것 재접근, 공간 지역성 = 인접 주소 접근. 둘의 정의가 뒤바뀐 보기가 단골 함정.
