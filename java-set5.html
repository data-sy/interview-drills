Q41객관식 · Object 메서드

모든 클래스가 Object로부터 물려받는 메서드가 아닌 것은?

  1. toString()
  2. equals()
  3. hashCode()
  4. compareTo()
정답 확인
④ compareTo()
compareTo()는 Comparable 인터페이스의 메서드로 Object에는 없다. toString/equals/hashCode는 Object가 기본 제공하므로 재정의를 통해 로그 출력과 동등성 비교의 품질을 높인다.
Q42객관식 · Integer 캐시

Integer a=127, b=127; Integer c=128, d=128; 일 때 a==b와 c==d의 결과는?

  1. 둘 다 true
  2. 둘 다 false
  3. a==b는 true, c==d는 false
  4. a==b는 false, c==d는 true
정답 확인
③ a==b는 true, c==d는 false
Integer는 -128~127 범위를 캐시해 같은 객체를 재사용하므로 127은 ==가 true이지만, 128은 새 객체가 생겨 false다. 그래서 래퍼 타입 비교는 반드시 equals로 해야 금액·수량 비교 버그를 피한다.
Q43객관식 · protected

protected 접근제어자에 대한 설명으로 옳은 것은?

  1. 같은 클래스에서만 접근할 수 있다
  2. 같은 패키지와, 패키지가 달라도 상속받은 자식 클래스에서 접근할 수 있다
  3. 어디서나 접근할 수 있다
  4. 자식 클래스에서는 접근할 수 없다
정답 확인
② 같은 패키지와, 패키지가 달라도 상속받은 자식 클래스에서 접근할 수 있다
protected는 같은 패키지 전체와, 패키지가 다르더라도 상속받은 자식 클래스에서 접근된다. 프레임워크가 확장 지점을 protected로 열어 두는 이유다.
Q44객관식 · 초기화 순서

클래스가 처음 로딩되고 인스턴스가 생성될 때의 초기화 실행 순서로 옳은 것은?

  1. static 블록(최초 1회) → 인스턴스 블록 → 생성자
  2. 생성자 → static 블록 → 인스턴스 블록
  3. 인스턴스 블록 → 생성자 → static 블록
  4. 생성자만 실행된다
정답 확인
① static 블록(최초 1회) → 인스턴스 블록 → 생성자
static 초기화는 클래스 로딩 시 딱 한 번 실행되고, 이후 객체를 만들 때마다 인스턴스 블록 → 생성자 순으로 실행된다. 공용 설정 로딩을 static에 두는 근거이며 이 순서를 오해하면 초기화 버그가 난다.
Q45객관식 · 다형성 실무(매개변수)

메서드 파라미터를 구체 타입 ArrayList 대신 상위 타입 List로 받는 것의 이점은?

  1. 실행 속도가 향상된다
  2. 컴파일이 빨라진다
  3. 구현 교체가 쉬워지고 결합도가 낮아진다
  4. 메모리 사용량이 줄어든다
정답 확인
③ 구현 교체가 쉬워지고 결합도가 낮아진다
상위 타입(인터페이스)에 의존하면 LinkedList 등 다른 구현으로 갈아끼워도 호출부를 고치지 않아도 된다. "구현이 아닌 인터페이스에 프로그래밍하라"는 원칙으로 SI 코드의 유지보수성을 높인다.
Q46객관식 · Iterator

Iterator에 대한 설명으로 옳은 것은?

  1. 컬렉션을 순회하며 hasNext()/next()로 요소에 접근하는 표준 방법이다
  2. 컬렉션의 크기를 두 배로 늘린다
  3. Map만 순회할 수 있다
  4. 역방향 순회만 지원한다
정답 확인
① 컬렉션을 순회하며 hasNext()/next()로 요소에 접근하는 표준 방법이다
Iterator는 컬렉션 종류와 무관하게 일관된 순회 인터페이스를 제공하고 순회 중 안전한 remove()도 지원한다. 여러 종류의 컬렉션을 공통 코드로 처리하는 배치 작업에서 유용하다.
Q47객관식 · 배열 vs ArrayList

배열(int[])과 ArrayList의 차이로 옳은 것은?

  1. 배열은 크기가 가변이고 ArrayList는 고정이다
  2. 배열은 크기가 고정이고 ArrayList는 동적으로 크기가 늘어난다
  3. 배열에는 기본형을 담을 수 없다
  4. ArrayList는 인덱스 접근이 불가능하다
정답 확인
② 배열은 크기가 고정이고 ArrayList는 동적으로 크기가 늘어난다
배열은 생성 시 크기가 고정되지만 ArrayList는 내부 배열을 재할당하며 자동으로 확장된다. 조회 결과처럼 개수를 미리 알 수 없는 데이터를 담기에는 ArrayList가 편하다.
Q48코드읽기 · 문자열 연결

다음 코드의 출력값을 쓰시오.

String s = "1" + 2 + 3;
int n = 1 + 2 + "3".length();
System.out.println(s + " / " + n);
모범답안 확인
모범답안 · 123 / 4
s는 왼쪽부터 문자열 연결이 일어나 "123"이 되고, n은 1 + 2 + ("3"의 길이 1) = 4다. 문자열과 숫자가 섞인 + 연산의 우선순위를 착각하면 화면 표시 값이 틀어지는 버그가 난다.
Q49단답 · Comparable vs Comparator

Comparable과 Comparator의 차이를 한 줄로 쓰시오.

모범답안 확인
모범답안
Comparable은 클래스 자신이 compareTo로 기본 정렬 기준을 정의하고, Comparator는 외부에서 compare로 별도의(상황별) 정렬 기준을 제공한다.

하나의 자연스러운 기본 정렬은 Comparable, 이름순·날짜순처럼 상황별 다중 정렬은 Comparator로 구현한다. 고객 요구별 정렬 화면을 만들 때 자주 쓴다.
Q50서술형 · 인터페이스 느슨한 결합

인터페이스를 사용해 모듈 간 결합도를 낮추는 방식과 그 실무적 이점을 설명하시오.

모범답안 확인
모범답안
호출부가 구현 클래스가 아니라 인터페이스(계약)에 의존하게 하고, 실제 구현체는 외부에서 주입(DI)받는다. 그러면 구현을 교체·추가해도 호출부를 수정하지 않고 확장할 수 있으며, 테스트할 때 가짜 구현(Mock)으로 대체하기도 쉽다.

채점 포인트는 인터페이스 의존, 구현 주입(교체 용이), 테스트 용이성이다. 스프링 DI의 근간이며 요구 변경이 잦은 SI에서 변경 비용을 크게 줄인다.

이 세트 우선복습 3줄

  1. Integer 캐시(-128~127) — 이 범위는 ==가 true, 벗어나면(128) false. 래퍼 비교는 항상 equals로.
  2. 초기화 순서 — static 블록(최초 1회) → 인스턴스 블록 → 생성자. static은 클래스 로딩 시 딱 한 번만 실행된다.
  3. 문자열 + 연산 — 왼쪽부터 순서대로 평가. "1"+2+3은 "123", 숫자끼리 먼저면 덧셈. 혼합 시 우선순위 착각 주의.
