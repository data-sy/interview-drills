Q41구두 · 오버로딩 vs 오버라이딩

오버로딩과 오버라이딩의 차이를 설명해보세요.

모범답변 요지
  • 오버로딩(overloading): 같은 이름, 다른 매개변수(개수·타입)로 여러 메서드 정의(컴파일 시점)
  • 오버라이딩(overriding): 상속 관계에서 부모 메서드를 자식이 재정의(실행 시점, 시그니처 동일)
  • 오버로딩은 편의성, 오버라이딩은 다형성과 연결
  • 실무 연결: 이름은 비슷하지만 목적이 다른 개념이라, 면접에서 자주 헷갈리는 지점을 정확히 구분해 설명하는 것이 기본기 증명 포인트
Q42구두 · static

static(정적) 멤버가 무엇이고 사용할 때 주의할 점은 무엇인가요?

모범답변 요지
  • 인스턴스가 아니라 클래스에 속해, 객체 생성 없이 클래스명으로 접근
  • 모든 인스턴스가 공유하는 하나의 값(공용 상수·유틸리티에 적합)
  • 남용 시 전역 상태가 되어 테스트·동시성에서 문제, 인스턴스 멤버 직접 접근 불가
  • 실무 연결: 웹 애플리케이션은 다중 스레드로 static 변수를 공유하므로, 가변 static에 요청별 데이터를 담으면 데이터 섞임 사고가 남을 강조
Q43구두 · 코드리뷰

코드리뷰를 왜 한다고 생각하며, 리뷰할 때/받을 때 어떤 태도가 좋을까요?

모범답변 요지
  • 목적: 버그 조기 발견, 지식 공유, 코드 품질·일관성 유지
  • 리뷰할 때는 사람이 아닌 코드에 대해, 근거와 대안을 함께 제시
  • 받을 때는 방어적이지 않게 수용하고 배우는 자세
  • 실무 연결: 담당자가 바뀌어도 읽히는 코드가 중요한 SI에서 리뷰는 품질과 팀 학습의 수단이라, 감정 아닌 개선 중심 문화를 강조
Q44구두 · NULL 처리(SQL)

SQL에서 NULL을 다룰 때 조심할 점을 설명해보세요.

모범답변 요지
  • NULL은 '값이 없음(미지)'이라 = NULL이 아니라 IS NULL로 비교
  • 산술·문자열 연결에서 NULL이 섞이면 결과가 NULL, 집계함수는 NULL을 제외
  • COALESCE/NVL로 기본값 치환, NOT IN에 NULL 있으면 예상 밖 결과
  • 실무 연결: 통계 집계·조건 필터에서 NULL을 방치하면 건수·합계가 틀어져 잘못된 리포트가 나가므로, NULL 처리 규칙 준수가 데이터 신뢰성의 기본임을 강조
Q45구두 · HTTP vs HTTPS

HTTP와 HTTPS의 차이를 SSL/TLS와 함께 설명해보세요.

모범답변 요지
  • HTTP는 평문 통신이라 도청·변조에 취약
  • HTTPS는 SSL/TLS로 암호화해 기밀성·무결성·서버 인증을 제공
  • 인증서 기반으로 통신 상대 신뢰를 확인하고 데이터를 암호화
  • 실무 연결: 개인정보·로그인을 다루는 고객사 시스템은 HTTPS가 필수이며, 평문 전송은 곧 보안 사고이자 규제 위반이 됨을 강조
Q46구두 · 보안/개인정보

개인정보나 민감한 데이터를 다룰 때 어떤 점을 지켜야 한다고 생각하나요?

모범답변 요지
  • 최소 수집·최소 권한 원칙, 필요한 사람만 접근
  • 비밀번호·주민번호 등은 암호화(해시)·마스킹 처리
  • 로그·화면·URL에 민감정보 노출 금지, 전송은 HTTPS
  • 실무 연결: 고객 데이터를 다루는 SI는 유출 사고가 법적 책임·신뢰 붕괴로 직결되므로, 개발 단계부터 보안을 습관화하는 태도를 강조
Q47구두 · 교착상태

교착상태(deadlock)가 무엇이고 왜 발생하는지 설명해보세요.

모범답변 요지
  • 둘 이상의 작업이 서로가 가진 자원을 기다리며 무한 대기하는 상태
  • 발생 4조건: 상호배제, 점유와 대기, 비선점, 순환 대기
  • 자원 획득 순서 통일 등으로 예방 가능
  • 실무 연결: DB에서 여러 트랜잭션이 서로의 락을 기다리며 데드락이 나는 일이 흔해, 갱신 순서를 일관되게 잡는 것이 실무 예방책임을 언급
Q48구두 · 가비지 컬렉션

자바의 가비지 컬렉션(GC)이 무엇이고 개발자가 알아둘 점은 무엇인가요?

모범답변 요지
  • 더 이상 참조되지 않는 객체(힙)를 JVM이 자동으로 회수해 메모리 관리
  • 개발자가 직접 free 하지 않아도 되지만, 참조를 계속 붙들면(컬렉션 누적 등) 메모리 누수 발생
  • GC가 도는 동안 성능 영향(stop-the-world)이 있을 수 있음
  • 실무 연결: 장기간 무중단 운영되는 SI 서버에서 참조 해제를 놓친 누수가 서서히 OOM으로 이어지므로, 자동 관리라도 참조 관리는 개발자 몫임을 강조
Q49구두 · 로그/디버깅

재현되지 않는 오류를 운영 환경에서 추적해야 한다면 어떻게 접근하겠습니까?

모범답변 요지
  • 로그·에러 메시지·스택트레이스로 발생 시점·조건을 좁힘
  • 언제·누구·어떤 데이터에서 발생하는지 패턴을 찾아 재현 조건 확보
  • 필요 시 로그 레벨을 조정해 정보 보강, 가설을 세워 하나씩 검증
  • 실무 연결: 운영 장애는 개발 환경에서 재현이 어려운 경우가 많아, 평소 의미 있는 로그를 남기는 습관이 추적 속도를 좌우함을 강조
Q50구두 · 성장 계획

입사 후 어떤 개발자로 성장하고 싶은지 계획을 말해보세요.

모범답변 요지
  • 단기: 회사 업무·도메인·기술 스택을 익혀 1인분 몫을 하는 것
  • 중기: 특정 분야(백엔드·DB 등) 전문성을 키우고 후배를 도울 수 있는 수준
  • 장기: 기술과 도메인 이해를 겸비해 문제를 주도적으로 해결하는 개발자
  • 실무 연결: 막연한 포부가 아니라 회사가 하는 SI 업무와 연결된 단계적 계획을 제시해, 오래 함께 성장할 사람이라는 신뢰를 주는 것이 핵심

이 세트 우선복습 3줄

  1. 오버로딩 vs 오버라이딩 — 오버로딩은 같은 이름·다른 매개변수(컴파일 시점), 오버라이딩은 상속에서 시그니처 동일하게 재정의(실행 시점). 이름만 비슷할 뿐 목적이 다르다.
  2. 가변 static의 함정 — 웹은 다중 스레드로 static을 공유하므로, 요청별 데이터를 가변 static에 담으면 데이터가 섞인다. static은 공용 상수·유틸리티에만.
  3. SQL NULL 비교 — = NULL은 항상 거짓, IS NULL로 비교. 집계는 NULL을 제외하고 NOT IN에 NULL이 섞이면 결과가 비므로 COALESCE로 방어.
