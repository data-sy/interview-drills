Q31객관식 · 그래프

그래프(Graph)를 컴퓨터에 표현하는 두 가지 대표 방식은?

  1. 인접 행렬(Adjacency Matrix)과 인접 리스트(Adjacency List)
  2. 스택과 큐
  3. 전위 순회와 후위 순회
  4. 해시와 정렬
정답 확인
① 인접 행렬(Adjacency Matrix)과 인접 리스트(Adjacency List)
그래프는 정점 간 연결을 2차원 배열로 담는 인접 행렬, 각 정점의 이웃을 리스트로 담는 인접 리스트로 표현한다. 간선이 적으면(희소) 인접 리스트가 메모리에 유리하다. SI에서 조직 관계·연계 흐름·의존성 등을 그래프로 모델링할 때 표현 방식 선택이 성능에 영향을 준다.
Q32객관식 · 정렬 안정성

정렬 알고리즘의 "안정성(Stability)"에 대한 설명으로 옳은 것은?

  1. 정렬 속도가 항상 일정하다는 뜻이다
  2. 같은 값을 가진 원소들의 원래 순서가 정렬 후에도 유지된다는 뜻이다
  3. 메모리를 추가로 쓰지 않는다는 뜻이다
  4. 항상 최선의 시간복잡도를 보장한다는 뜻이다
정답 확인
② 같은 값을 가진 원소들의 원래 순서가 정렬 후에도 유지된다는 뜻이다
안정 정렬은 동일 키를 가진 원소들의 입력 순서를 정렬 후에도 보존한다(병합 정렬은 안정, 퀵 정렬은 불안정). SI에서 이미 부서순 정렬된 목록을 다시 이름순으로 정렬할 때 안정 정렬이면 같은 이름 내 부서 순서가 유지되어 다중 기준 정렬에 유리하다.
Q33객관식 · CPU 스케줄링

CPU 스케줄링 기법 중 "먼저 도착한 프로세스를 먼저 처리"하는 비선점 방식은?

  1. Round Robin
  2. SJF(Shortest Job First)
  3. FCFS(First Come First Served)
  4. Priority Scheduling
정답 확인
③ FCFS(First Come First Served)
FCFS는 도착 순서대로 처리하는 비선점 스케줄링으로 구현이 간단하나, 긴 작업이 앞에 오면 뒤가 오래 기다리는 콘보이 효과가 있다. Round Robin은 시간 할당량으로 돌아가는 선점 방식이다. SI 배치 작업 스케줄 설계 시 처리 순서·우선순위 정책 선택의 기초 개념이다.
Q34객관식 · 사설 IP·서브넷

사설 IP(Private IP) 주소에 대한 설명으로 옳은 것은?

  1. 인터넷에 직접 라우팅되어 외부에서 바로 접근할 수 있다
  2. 전 세계에서 유일하게 하나만 존재하도록 할당된다
  3. 192.0.2.x 대역이 대표적인 사설 IP 범위다
  4. 10.x.x.x·172.16~31.x.x·192.168.x.x 대역이며 내부망에서만 쓰이고 NAT를 거쳐 외부와 통신한다
정답 확인
④ 10.x.x.x·172.16~31.x.x·192.168.x.x 대역이며 내부망에서만 쓰이고 NAT를 거쳐 외부와 통신한다
사설 IP는 조직 내부망 전용 대역(10/8, 172.16/12, 192.168/16)으로 인터넷에 직접 라우팅되지 않고 NAT(공인 IP 변환)를 통해 외부와 통신한다. 전 세계 유일한 것은 공인 IP다. SI에서 고객사 폐쇄망·내부 서버의 IP 대역과 방화벽/NAT 정책을 파악하는 것이 연계 구축·장애 대응의 기본이 된다.
Q35객관식 · 제3정규형

제3정규형(3NF)이 제거하려는 함수 종속은?

  1. 부분 함수 종속
  2. 이행적(전이) 함수 종속
  3. 다치 종속
  4. 조인 종속
정답 확인
② 이행적(전이) 함수 종속
3NF는 2NF를 만족하면서 기본키가 아닌 속성이 다른 비기본키 속성에 종속되는 이행적 종속을 제거한다(부분 종속 제거는 2NF). SI 데이터 모델링에서 코드값→코드명 같은 종속을 별도 코드 테이블로 분리하는 것이 이행적 종속 제거의 전형적 사례다.
Q36객관식 · 힙(Heap)

우선순위 큐(Priority Queue)를 효율적으로 구현하는 데 가장 적합한 자료구조는?

  1. 힙(Heap)
  2. 일반 배열(정렬 없음)
  3. 단일 연결리스트
  4. 해시테이블
정답 확인
① 힙(Heap)
힙은 최댓값(또는 최솟값)을 O(1)로 조회하고 삽입/삭제를 O(log n)으로 처리하는 완전 이진 트리라 우선순위 큐에 최적이다. SI에서 긴급도 높은 작업을 먼저 처리하는 작업 스케줄러, 다익스트라 최단경로 계산 등에 활용된다.
Q37객관식 · 컨텍스트 스위칭

컨텍스트 스위칭(Context Switching)에 대한 설명으로 옳은 것은?

  1. 단일 스레드 프로그램에서만 발생한다
  2. 프로세스를 디스크에서 완전히 삭제하는 작업이다
  3. 오버헤드가 전혀 없어 자주 할수록 성능이 좋아진다
  4. CPU가 실행 중인 프로세스/스레드를 전환할 때 상태를 저장·복원하는 과정이다
정답 확인
④ CPU가 실행 중인 프로세스/스레드를 전환할 때 상태를 저장·복원하는 과정이다
컨텍스트 스위칭은 현재 실행 문맥(레지스터·PC 등)을 PCB에 저장하고 다음 것을 복원하는 과정으로, 잦으면 오버헤드가 커진다. SI 서버에서 스레드를 과도하게 늘리면 스위칭 비용으로 오히려 처리량이 떨어지므로 적정 스레드 풀 크기 설정이 중요하다.
Q38객관식 · 캐시 히트/미스

총 1000회 메모리 접근 중 950회가 캐시에서 처리되었다면 캐시 적중률(Hit Ratio)은?

  1. 5%
  2. 50%
  3. 95%
  4. 99%
정답 확인
③ 95%
적중률 = 캐시 적중 횟수 / 전체 접근 = 950/1000 = 95%다. 적중률이 높을수록 느린 하위 메모리 접근이 줄어 평균 접근 시간이 짧아진다. SI에서 캐시 적중률 지표를 모니터링해 캐시 크기·만료 정책을 튜닝하면 응답 성능을 개선할 수 있다.
Q39단답 · 이진 탐색

정렬된 n개의 데이터에 대한 이진 탐색(Binary Search)의 시간복잡도를 Big-O로 쓰시오.

모범답안 확인
모범답안 · O(log n)
매 단계 탐색 범위를 절반으로 줄이므로 최대 log₂n번 비교하면 되어 O(log n)이다. 선형 탐색 O(n)보다 대량 데이터에서 훨씬 빠르다. SI에서 정렬 인덱스가 잡힌 컬럼 조회가 빠른 이유가 바로 이 로그 시간 탐색(B-tree 인덱스) 덕분이다.
Q40서술형 · 블로킹/논블로킹

블로킹(Blocking)과 논블로킹(Non-blocking)의 차이를 설명하시오.

모범답안 확인
모범답안
블로킹은 호출한 함수가 작업을 끝낼 때까지 제어권을 돌려주지 않아 호출한 쪽이 그 자리에서 멈춰 대기하는 방식이다. 논블로킹은 작업 완료 여부와 무관하게 즉시 제어권을 돌려주어, 호출한 쪽이 대기하지 않고 다른 일을 계속하다가 나중에 결과를 확인(폴링/콜백)하는 방식이다.

채점 포인트는 "제어권 반환 시점(대기 여부)"이다. 동기/비동기는 결과 통지 방식, 블로킹/논블로킹은 제어권 관점이라는 구분도 언급하면 가점. SI에서 외부 연계 호출을 논블로킹으로 처리하면 응답 지연 중에도 다른 요청을 처리해 처리량을 유지할 수 있다.

이 세트 우선복습 3줄

  1. 정렬 안정성 — 같은 키 원소의 원래 순서가 보존되면 안정 정렬(병합=안정, 퀵=불안정). 다중 기준 정렬의 근거.
  2. 사설 IP 대역 + NAT — 10/8·172.16~31·192.168 은 내부망 전용, 인터넷 직접 라우팅 불가라 NAT로 공인 IP 변환을 거친다.
  3. 동기/비동기 vs 블로킹/논블로킹 — 전자는 "결과 통지 방식", 후자는 "제어권 반환 시점". 두 축을 헷갈리지 말 것.
