Q21객관식 · 추상클래스 vs 인터페이스

추상클래스와 인터페이스의 차이로 옳은 것은?

  1. 인터페이스는 다중 구현이 가능하나 클래스는 단일 상속만 된다
  2. 추상클래스는 곧바로 인스턴스화할 수 있다
  3. 인터페이스는 생성자를 가진다
  4. 추상클래스에는 추상 메서드를 둘 수 없다
정답 확인
① 인터페이스는 다중 구현이 가능하나 클래스는 단일 상속만 된다
클래스는 하나만 상속(extends)하지만 인터페이스는 여러 개 구현(implements)할 수 있다. 공통 뼈대는 추상클래스로, 능력(계약)의 조합은 인터페이스로 나눠 유연한 모듈을 설계한다.
Q22객관식 · 인터페이스 default 메서드

자바 8부터 인터페이스에 추가된 기능은?

  1. 인스턴스 필드 상태 저장
  2. 구현부를 가진 default 메서드
  3. 생성자
  4. private 인스턴스 필드
정답 확인
② 구현부를 가진 default 메서드
default 메서드로 기존 구현체를 깨지 않고 인터페이스에 새 기능을 확장할 수 있게 됐다. 운영 중인 고객사 공용 인터페이스에 하위호환을 유지하며 기능을 추가할 때 유용하다.
Q23객관식 · ArrayList vs LinkedList

리스트 중간의 삽입/삭제가 매우 빈번하고 인덱스 임의 접근이 드문 경우 유리한 것은?

  1. ArrayList
  2. 둘의 성능이 동일하다
  3. LinkedList
  4. 기본 배열
정답 확인
③ LinkedList
LinkedList는 노드 연결 구조라 위치를 알면 삽입/삭제가 O(1)이지만 임의 접근은 O(n)이고, ArrayList는 그 반대다. 데이터 접근 특성에 맞춰 골라야 대용량 처리 성능을 확보한다.
Q24객관식 · HashMap

HashMap의 특성으로 옳은 것은?

  1. 키 중복을 허용한다
  2. 저장 순서를 항상 보장한다
  3. null 키를 절대 허용하지 않는다
  4. 키는 중복 불가, 값은 중복 가능하며 순서는 보장하지 않는다
정답 확인
④ 키는 중복 불가, 값은 중복 가능하며 순서는 보장하지 않는다
HashMap은 키가 유일하고 값은 중복 가능하며 순서를 보장하지 않는다(하나의 null 키 허용). 코드-명칭 매핑 조회에 자주 쓰고, 입력 순서가 필요하면 LinkedHashMap을 쓴다.
Q25객관식 · HashSet

HashSet에 이미 있는 값과 같은 값을 다시 add하면?

  1. 중복은 저장되지 않아 하나만 남는다
  2. 예외가 발생한다
  3. 두 개가 저장된다
  4. 마지막 값으로 덮어쓴다
정답 확인
① 중복은 저장되지 않아 하나만 남는다
Set은 중복을 허용하지 않아 add가 false를 반환하고 요소는 하나만 남는다. 중복 고객 ID 제거 같은 집합 연산에 그대로 활용한다.
Q26객관식 · Generics

제네릭(List<String>)을 사용하는 주된 이점은?

  1. 실행 속도가 빨라진다
  2. 컴파일 시점 타입 안전성 확보와 형변환 코드 제거
  3. 메모리를 절반으로 줄인다
  4. 다중 상속을 가능하게 한다
정답 확인
② 컴파일 시점 타입 안전성 확보와 형변환 코드 제거
제네릭은 컴파일 시점에 타입을 검증해 ClassCastException을 예방하고 캐스팅 코드를 없앤다. 공통 유틸·DAO를 타입 안전하게 재사용해 코드 품질을 높인다.
Q27객관식 · final 세 용법

final 키워드의 사용 위치별 의미로 틀린 것은?

  1. final 변수: 재할당 불가
  2. final 메서드: 오버라이딩 불가
  3. final 클래스: 상속 불가
  4. final 메서드: 오버로딩 불가
정답 확인
④ final 메서드: 오버로딩 불가
final 메서드는 오버라이딩(재정의)만 막을 뿐 오버로딩과는 무관하다. 상속으로 바꾸면 안 되는 보안·핵심 로직을 final로 잠가 무분별한 변경을 막는다.
Q28코드읽기 · List 순회 중 삭제

다음 코드를 실행할 때 발생하는 것을 쓰시오.

List<Integer> list = new ArrayList<>(List.of(1, 2, 3));
for (Integer n : list) {
    if (n == 2) list.remove(n);
}
모범답안 확인
모범답안 · ConcurrentModificationException 발생
for-each(내부 Iterator) 순회 중 컬렉션을 직접 remove하면 modCount 불일치로 예외가 난다. Iterator.remove()나 removeIf를 써야 안전하며, 배치 데이터 정제 로직에서 흔한 함정이다.
Q29단답 · StringBuilder vs StringBuffer

StringBuilder와 StringBuffer의 핵심 차이를 한 줄로 쓰시오.

모범답안 확인
모범답안
StringBuffer는 메서드가 동기화(synchronized)되어 스레드 안전하지만 느리고, StringBuilder는 동기화가 없어 단일 스레드에서 더 빠르다.

둘 다 가변 문자열이다. 대부분의 단일 스레드 리포트 생성에는 StringBuilder가 적합하고, 여러 스레드가 공유하는 상황에만 StringBuffer를 고른다.
Q30서술형 · ArrayList vs LinkedList 선택

실무에서 ArrayList와 LinkedList를 각각 언제 선택하는지 시간복잡도 근거를 들어 설명하시오.

모범답안 확인
모범답안
조회·인덱스 접근이 잦으면 임의 접근이 O(1)인 ArrayList를 쓴다. 리스트 앞이나 중간의 삽입·삭제가 매우 빈번하면 그 연산이 O(1)인 LinkedList가 유리하다. 다만 실무 대부분은 조회 위주라 특별한 이유가 없으면 기본은 ArrayList를 택한다.

채점 포인트는 두 자료구조의 접근 vs 삽입/삭제 복잡도 대비와, 현실적으로 기본은 ArrayList라는 결론이다. 근거 없이 LinkedList를 선호하면 감점이다.

이 세트 우선복습 3줄

  1. 추상클래스 vs 인터페이스 — 클래스는 단일 상속(extends), 인터페이스는 다중 구현(implements). 공통 뼈대는 추상클래스, 능력의 조합은 인터페이스.
  2. 순회 중 삭제 함정 — for-each로 돌면서 list.remove()하면 ConcurrentModificationException. Iterator.remove()나 removeIf로 안전하게.
  3. ArrayList vs LinkedList — ArrayList는 임의 접근 O(1), LinkedList는 위치를 알 때 삽입/삭제 O(1). 근거 없으면 기본은 ArrayList.
