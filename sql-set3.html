Q21객관식 · 상관 서브쿼리

상관(correlated) 서브쿼리에 대한 설명으로 옳은 것은?

  1. 서브쿼리가 바깥 쿼리와 무관하게 한 번만 실행된다
  2. 서브쿼리가 바깥 쿼리의 컬럼을 참조해 행마다 반복 실행된다
  3. 반드시 FROM 절에만 쓸 수 있다
  4. 집계함수를 쓸 수 없다
정답 확인
② 서브쿼리가 바깥 쿼리의 컬럼을 참조해 행마다 반복 실행된다
상관 서브쿼리는 외부 행의 값을 받아 행마다 평가되어 EXISTS·행별 비교에 쓰인다. "각 사원의 급여가 자기 부서 평균보다 높은가" 같은 SI 요건에 유용하지만, 대량 데이터에서는 반복 실행 비용이 커 튜닝 대상이 된다.
Q22객관식 · 인덱스 개념

인덱스(INDEX)에 대한 설명으로 옳지 않은 것은?

  1. 조회(SELECT) 성능을 높이기 위한 자료구조다
  2. INSERT/UPDATE/DELETE 시 인덱스도 갱신되어 쓰기 부하가 생긴다
  3. 인덱스를 많이 만들수록 모든 작업이 항상 빨라진다
  4. 카디널리티가 높은(값이 다양한) 컬럼에 대체로 효과적이다
정답 확인
③ 인덱스를 많이 만들수록 모든 작업이 항상 빨라진다
인덱스는 조회를 빠르게 하지만 쓰기마다 유지비용이 들고 저장공간도 쓴다. SI에서 배치 INSERT가 느려지는 원인이 과도한 인덱스인 경우가 많아, 조회 패턴에 맞춰 꼭 필요한 인덱스만 설계하는 것이 실무 튜닝의 기본이다.
Q23객관식 · 문자열 함수

오라클에서 SUBSTR('20260707', 5, 2) 의 결과는?

  1. '2026'
  2. '0707'
  3. '26'
  4. '07'
정답 확인
④ '07'
SUBSTR(문자열, 시작위치, 길이)는 5번째 문자부터 2글자를 반환하므로 '07'(월)이다. SI에서 YYYYMMDD 형태 코드성 문자열에서 연·월·일을 잘라 쓰는 일이 흔해 시작위치가 1부터 시작함을 정확히 기억해야 한다.
Q24객관식 · 날짜 함수

오늘 날짜를 반환하며, 여기에 +1을 하면 내일이 되는(오라클) 함수/표현은?

  1. SYSDATE
  2. NOW()
  3. GETDATE()
  4. CURRENT_TIMESTAMP()의 문자열
정답 확인
① SYSDATE
오라클은 SYSDATE가 현재 날짜/시각을 DATE 타입으로 주며 산술로 SYSDATE+1은 하루 뒤가 된다(MySQL은 NOW, SQL Server는 GETDATE). SI 배치의 "어제 데이터 조회", "N일 경과 미처리 건" 조건에 자주 쓰이는데 DBMS별 함수명이 달라 이식 시 주의한다.
Q25객관식 · 제약조건 FK

외래키(FOREIGN KEY) 제약의 역할로 옳은 것은?

  1. 해당 컬럼 값이 반드시 유일하도록 강제한다
  2. 참조하는 테이블에 없는 값이 입력되지 못하게 참조무결성을 지킨다
  3. 컬럼에 NULL이 들어가지 못하게 막는다
  4. 컬럼의 기본값을 자동으로 지정한다
정답 확인
② 참조하는 테이블에 없는 값이 입력되지 못하게 참조무결성을 지킨다
FK는 자식 테이블 값이 부모 테이블의 존재하는 키만 참조하도록 보장한다. SI에서 주문의 고객ID가 실제 고객 테이블에 있어야 한다는 규칙을 DB 차원에서 지켜, 애플리케이션 버그로 인한 고아 데이터(orphan)를 방지한다.
Q26객관식 · 트랜잭션 COMMIT/ROLLBACK

여러 UPDATE를 실행한 뒤 오류가 나서 모든 변경을 취소하려 한다. 사용할 명령은?

  1. COMMIT
  2. SAVEPOINT
  3. GRANT
  4. ROLLBACK
정답 확인
④ ROLLBACK
ROLLBACK은 아직 커밋되지 않은 트랜잭션의 변경을 모두 취소해 이전 상태로 되돌린다(원자성). SI 정산·이체처럼 여러 테이블을 함께 바꾸는 작업은 "모두 성공 또는 모두 취소"가 필수라 트랜잭션으로 묶고 실패 시 ROLLBACK한다.
Q27쿼리작성 · 서브쿼리(스칼라/인라인)

아래 스키마에서 요구사항을 만족하는 SQL을 작성하라.
스키마: EMP(emp_id, name, salary)
요구사항: 전체 사원의 평균 급여보다 높은 급여를 받는 사원의 name과 salary를 조회하라.

모범답안 확인
모범답안
SELECT name, salary
FROM   EMP
WHERE  salary > (SELECT AVG(salary) FROM EMP);
서브쿼리로 평균이라는 단일 값을 구해 WHERE에서 비교하는 스칼라 서브쿼리 패턴이다. "평균/기준치 초과 대상 추출"은 SI 통계 요건의 단골이며, 집계값과 개별 행을 한 번에 비교할 수 없어 서브쿼리로 분리해야 한다.
Q28쿼리작성 · INSERT

아래 스키마에서 요구사항을 만족하는 SQL을 작성하라.
스키마: DEPT(dept_id, dept_name, location)
요구사항: dept_id=50, dept_name='품질관리', location='서울' 인 부서를 한 건 추가하라.

모범답안 확인
모범답안
INSERT INTO DEPT (dept_id, dept_name, location)
VALUES (50, '품질관리', '서울');
컬럼 목록을 명시하고 VALUES 순서를 맞추는 것이 채점 포인트다. 컬럼을 생략하면 테이블 정의 순서에 의존해 유지보수 중 컬럼이 추가되면 깨지므로, SI 운영 스크립트에서는 항상 컬럼명을 명시하는 것이 안전하다.
Q29쿼리작성 · CASE WHEN

아래 스키마에서 요구사항을 만족하는 SQL을 작성하라.
스키마: EMP(emp_id, name, salary)
요구사항: 각 사원의 name과 함께, salary가 5000 이상이면 '상', 3000 이상이면 '중', 그 외 '하'로 등급(grade)을 함께 출력하라.

모범답안 확인
모범답안
SELECT name,
       CASE WHEN salary >= 5000 THEN '상'
            WHEN salary >= 3000 THEN '중'
            ELSE '하'
       END AS grade
FROM   EMP;
CASE 조건을 큰 구간부터 배치하고 END에 별칭을 붙이는 것이 핵심이다. 금액/점수를 구간 라벨로 변환하는 것은 SI 대시보드·통계 리포트에서 매우 흔하며, 조건 순서를 뒤집으면 모두 '중'으로 잡히는 오류가 난다.
Q30단답 · GROUP BY/COUNT

EMP 테이블의 dept_id 값이 10, 10, 20, 20, 20, 30 (총 6행)이다. SELECT dept_id, COUNT(*) FROM EMP GROUP BY dept_id 의 결과는 몇 행인가? 숫자로 답하라.

모범답안 확인
모범답안 · 3
GROUP BY는 같은 값끼리 묶으므로 서로 다른 dept_id 개수(10·20·30)인 3행이 나온다. 그룹 결과 행수는 곧 "구별되는 그룹 수"라는 감각은 SI에서 카테고리·부서별 집계 결과 검증 시 유용하다.

이 세트 우선복습 3줄

  1. 인덱스 트레이드오프 — 조회는 빨라지지만 INSERT/UPDATE/DELETE마다 갱신 비용이 든다. "많이 만들수록 항상 빠르다"는 함정.
  2. 상관 서브쿼리 — 바깥 쿼리의 컬럼을 참조해 행마다 반복 실행된다. 부서별 기준값 비교에 강력하지만 대량 데이터에선 튜닝 대상.
  3. SUBSTR 시작위치 — SUBSTR(문자열, 시작, 길이)의 시작위치는 1부터. 0이나 배열식 0-based로 착각하면 한 칸씩 밀린다.
