상속·다형성·생성자부터 오버라이딩 vs 오버로딩까지

보기를 고른 뒤 채점하기를 누르면 정오답과 해설이 열립니다. 서술·단답·코드읽기는 직접 적어보고 모범답안과 비교해 스스로 채점하세요.

Q11객관식 · 상속

자바의 클래스 상속에 대한 설명으로 옳은 것은?

  1. 클래스 다중 상속을 지원한다
  2. 상위 클래스의 private 필드에 자식이 직접 접근한다
  3. extends 키워드로 단일 상속만 가능하다
  4. 생성자도 상속된다
정답 확인
③ extends 키워드로 단일 상속만 가능하다
자바는 클래스 다중 상속을 금지(인터페이스로 대체)하고 extends로 단일 상속만 허용한다. private은 상속돼도 직접 접근이 안 되고 생성자는 상속 대상이 아니다. 상속을 남용하면 결합도가 높아져 레거시 수정 시 파급이 커진다.
Q12객관식 · 다형성/동적바인딩

Animal a = new Dog(); 에서 a.sound()가 Dog의 sound()를 호출하게 하는 메커니즘은?

  1. 런타임 실제 객체 기준의 동적 바인딩
  2. 정적 바인딩
  3. 컴파일 에러
  4. 항상 Animal의 sound() 호출
정답 확인
① 런타임 실제 객체 기준의 동적 바인딩
오버라이딩된 메서드는 참조 타입이 아니라 런타임의 실제 객체 타입으로 호출이 결정된다(동적 바인딩). 덕분에 결제수단·인증방식 구현체만 갈아끼워 확장할 수 있어 SI에서 요구 변경 대응이 쉬워진다.
Q13객관식 · 생성자(기본생성자)

클래스에 생성자를 하나도 정의하지 않으면 어떻게 되는가?

  1. 컴파일 에러가 난다
  2. 상위 클래스 생성자를 그대로 복사한다
  3. 객체를 생성할 수 없다
  4. 컴파일러가 매개변수 없는 기본 생성자를 자동으로 추가한다
정답 확인
④ 컴파일러가 매개변수 없는 기본 생성자를 자동으로 추가한다
생성자를 하나라도 직접 정의하면 기본 생성자는 자동 생성되지 않는다. 매개변수 생성자만 만들어 놓고 new X()를 호출하다 나는 컴파일 오류가 신입 단골 실수다.
Q14객관식 · this/super

this()와 super()에 대한 설명으로 옳은 것은?

  1. 둘 다 생성자 첫 줄에 오며 한 생성자에서 동시에 쓸 수 있다
  2. super()는 상위 클래스 생성자를, this()는 같은 클래스의 다른 생성자를 호출한다
  3. this는 상위 클래스를, super는 자신을 가리킨다
  4. 메서드 어디서나 자유롭게 호출할 수 있다
정답 확인
② super()는 상위 클래스 생성자를, this()는 같은 클래스의 다른 생성자를 호출한다
this()/super() 호출은 생성자의 첫 문장에만 올 수 있고 한 생성자에서 둘을 함께 쓸 수 없다. 중복 초기화 코드를 this()로 모으면 필드가 늘어도 한 곳만 고쳐 유지보수가 쉬워진다.
Q15객관식 · static

static 멤버에 대한 설명으로 옳은 것은?

  1. 객체마다 별도로 생성된다
  2. 상속되지 않는다
  3. 객체 생성 없이 클래스명으로 접근할 수 있고 모든 인스턴스가 공유한다
  4. 생성자에서만 초기화된다
정답 확인
③ 객체 생성 없이 클래스명으로 접근할 수 있고 모든 인스턴스가 공유한다
static 멤버는 클래스 로딩 시 한 번 만들어져 모든 인스턴스가 공유하며 클래스명으로 접근한다. 공용 카운터·상수에 쓰지만 공유 상태이므로 멀티스레드 환경에서는 동기화 이슈를 조심해야 한다.
Q16객관식 · 오버로딩

메서드 오버로딩의 성립 조건으로 옳은 것은?

  1. 이름이 같고 매개변수의 개수/타입/순서가 달라야 한다
  2. 이름과 매개변수가 같고 반환형만 달라도 된다
  3. 반드시 상속 관계여야 한다
  4. 접근제어자가 서로 달라야 한다
정답 확인
① 이름이 같고 매개변수의 개수/타입/순서가 달라야 한다
오버로딩은 같은 이름에 다른 시그니처(매개변수 목록)를 요구하며 반환형만 다른 것은 성립하지 않는다. 다양한 입력을 같은 이름으로 받는 로그·포맷 유틸 설계에 흔히 쓴다.
Q17객관식 · String 불변성

String s = "a"; s = s + "b"; 실행 후에 대한 설명으로 옳은 것은?

  1. 기존 "a" 객체의 내용이 "ab"로 바뀐다
  2. s는 여전히 "a"이다
  3. 컴파일 에러가 난다
  4. String은 불변이라 새 객체 "ab"가 생성되고 s가 그것을 가리킨다
정답 확인
④ String은 불변이라 새 객체 "ab"가 생성되고 s가 그것을 가리킨다
String은 불변(immutable)이라 연산할 때마다 새 객체가 생긴다. 반복 문자열 연결이 성능 문제를 일으키므로 대량 리포트 생성 시 StringBuilder를 쓰는 근거가 된다.
Q18코드읽기 · 오버라이딩/동적바인딩

다음 코드의 출력값을 쓰시오.

class A { String who(){ return "A"; } }
class B extends A { String who(){ return "B"; } }

A x = new B();
System.out.println(x.who());
모범답안 확인
모범답안 · B
참조 타입은 A지만 실제 객체가 B이므로 오버라이딩된 B.who()가 호출된다(동적 바인딩). 인터페이스 타입 변수에 구현체를 갈아끼우는 SI 확장 패턴의 기본 원리다.
Q19단답 · == vs equals

참조형에서 == 연산자와 equals() 메서드가 비교하는 대상의 차이를 한 줄로 쓰시오.

모범답안 확인
모범답안
==는 두 참조가 같은 객체(주소)를 가리키는지 비교하고, equals()는 (재정의된 경우) 객체의 내용(값)이 같은지 비교한다.

문자열 비교를 ==로 하다 실패하는 것이 신입 단골 버그다. 고객 데이터의 동등성 판단은 반드시 equals로 해야 정확하다.
Q20서술형 · 오버라이딩 vs 오버로딩

오버라이딩과 오버로딩의 차이를 관계·시그니처·바인딩 시점 세 관점에서 설명하시오.

모범답안 확인
모범답안
오버라이딩은 상속 관계에서 상위 클래스 메서드를 자식이 같은 시그니처로 재정의하며 런타임(동적)에 바인딩된다. 오버로딩은 같은 클래스 안에서 같은 이름에 다른 매개변수로 여러 개를 두며 컴파일타임(정적)에 결정된다.

채점 포인트는 관계(상속 vs 동일 클래스)·시그니처(동일 vs 상이)·바인딩(런타임 vs 컴파일)이다. 이 구분이 프레임워크 확장 지점 설계의 기초가 된다.

이 세트 우선복습 3줄

  1. 기본 생성자 자동 생성 조건 — 생성자를 하나라도 직접 정의하면 매개변수 없는 기본 생성자는 자동 추가되지 않는다. new X() 컴파일 오류의 원인.
  2. 동적 바인딩 — Animal a = new Dog();에서 오버라이딩 메서드는 참조 타입이 아니라 런타임 실제 객체(Dog) 기준으로 호출된다.
  3. String 불변 + == vs equals — 문자열 연결마다 새 객체가 생기고, 값 비교는 반드시 equals()로. ==는 주소(같은 객체인지)만 본다.
