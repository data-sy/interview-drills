Q21객관식 · HTTP 메서드(PUT vs PATCH)

PUT과 PATCH의 차이로 옳은 것은?

  1. PUT은 리소스 전체 교체, PATCH는 일부 필드만 수정한다
  2. PUT은 부분 수정, PATCH는 전체 교체한다
  3. 둘 다 조회 전용 메서드다
  4. PATCH는 리소스를 삭제한다
정답 확인
① PUT은 리소스 전체 교체, PATCH는 일부 필드만 수정한다
PUT은 리소스 전체를 보낸 값으로 대체하고, PATCH는 변경할 일부 필드만 수정한다. 회원정보처럼 필드가 많은 자원을 부분 수정할 때 PATCH가 전송량·충돌을 줄이므로, SI에서 대용량 폼의 부분 저장 API를 설계할 때 구분이 중요하다.
Q22객관식 · 상태코드 401 vs 403

401과 403 상태코드의 차이로 옳은 것은?

  1. 401은 권한 부족, 403은 로그인 안 됨을 의미한다
  2. 401은 인증 안 됨(로그인 필요), 403은 인증됐으나 권한 없음을 의미한다
  3. 둘 다 서버 내부 오류다
  4. 401은 성공, 403은 리다이렉트다
정답 확인
② 401은 인증 안 됨(로그인 필요), 403은 인증됐으나 권한 없음을 의미한다
401 Unauthorized는 인증(로그인) 자체가 안 된 상태, 403 Forbidden은 인증은 됐지만 해당 자원 접근 권한이 없는 상태다. SI 시스템의 권한 등급(일반/관리자)별 접근 제어를 구현할 때, 미로그인은 401·권한 부족은 403으로 응답해야 프론트가 로그인 유도와 권한 안내를 구분할 수 있다.
Q23객관식 · HTTPS 대칭/비대칭 암호화

HTTPS(SSL/TLS) 통신에서 암호화 방식에 대한 설명으로 옳은 것은?

  1. 처음부터 끝까지 비대칭키만 사용해 본문을 암호화한다
  2. 대칭키로 인증서를 발급한다
  3. 핸드셰이크에서 비대칭키로 대칭키를 안전하게 교환하고, 이후 본문은 대칭키로 암호화한다
  4. 암호화 없이 인증서만 교환한다
정답 확인
③ 핸드셰이크에서 비대칭키로 대칭키를 안전하게 교환하고, 이후 본문은 대칭키로 암호화한다
비대칭키(공개키/개인키)는 안전하지만 느려 핸드셰이크 때 세션용 대칭키를 교환하는 데만 쓰고, 실제 대량 데이터 본문은 빠른 대칭키로 암호화한다. 이 하이브리드 방식을 이해해야 SI에서 성능과 보안을 함께 고려한 HTTPS·인증서 구성을 설명할 수 있다.
Q24객관식 · REST 제약·특징

REST 아키텍처의 특징으로 옳지 않은 것은?

  1. 자원을 URI로 식별하고 HTTP 메서드로 행위를 표현한다
  2. 서버는 클라이언트의 상태를 유지하지 않는 무상태를 지향한다
  3. 요청은 자기 서술적이며 표현(JSON/XML)으로 자원을 주고받는다
  4. 서버가 각 세션의 진행 상태를 반드시 저장해야 한다
정답 확인
④ 서버가 각 세션의 진행 상태를 반드시 저장해야 한다
REST는 무상태(Stateless)를 원칙으로 해 서버가 세션 상태를 저장하지 않는 것을 지향하므로 ④가 틀리다. 무상태·자원 중심 설계 덕분에 서버 확장과 캐싱이 쉬워, SI에서 외부 연계용 개방형 API를 만들 때 REST가 선호된다.
Q25객관식 · DOM

DOM(Document Object Model)에 대한 설명으로 옳은 것은?

  1. 서버가 DB를 조작하는 API다
  2. HTML 문서를 트리 구조의 객체로 표현해 JS로 접근·수정하게 하는 모델이다
  3. CSS 스타일을 압축하는 규칙이다
  4. HTTP 요청 헤더의 한 종류다
정답 확인
② HTML 문서를 트리 구조의 객체로 표현해 JS로 접근·수정하게 하는 모델이다
DOM은 브라우저가 HTML을 노드 트리 객체로 만들어, 자바스크립트가 요소를 찾고 내용·속성·스타일을 동적으로 바꿀 수 있게 한 모델이다. AJAX로 받은 데이터를 화면에 반영하는 것도 DOM 조작이며, SI의 동적 화면·유효성 검사 구현의 기초 개념이다.
Q26객관식 · 필터 vs 인터셉터

서블릿 필터(Filter)와 스프링 인터셉터(Interceptor)의 차이로 옳은 것은?

  1. 필터는 스프링 컨텍스트 안, 인터셉터는 그 바깥에서 동작한다
  2. 둘 다 DB 트랜잭션 전용 기능이다
  3. 필터는 서블릿 컨테이너 레벨, 인터셉터는 스프링 MVC(DispatcherServlet 이후) 레벨에서 동작한다
  4. 인터셉터는 요청을 가로챌 수 없다
정답 확인
③ 필터는 서블릿 컨테이너 레벨, 인터셉터는 스프링 MVC(DispatcherServlet 이후) 레벨에서 동작한다
필터는 서블릿 컨테이너 레벨에서 DispatcherServlet 앞뒤로 동작해 인코딩·전역 로깅 등에 쓰고, 인터셉터는 스프링 MVC 내부에서 컨트롤러 실행 전후에 동작해 로그인 체크·권한 검사에 쓴다. SI에서 공통 인증·인코딩 처리를 어느 계층에 둘지 판단할 때 이 차이가 기준이 된다.
Q27객관식 · 인증 vs 인가

인증(Authentication)과 인가(Authorization)의 차이로 옳은 것은?

  1. 인증은 권한 확인, 인가는 신원 확인이다
  2. 인증은 '누구인지' 신원 확인, 인가는 '무엇을 할 수 있는지' 권한 확인이다
  3. 둘은 같은 의미다
  4. 인가가 먼저 일어난 뒤 인증이 진행된다
정답 확인
② 인증은 '누구인지' 신원 확인, 인가는 '무엇을 할 수 있는지' 권한 확인이다
인증은 로그인처럼 사용자가 누구인지 확인하는 것이고, 인가는 인증된 사용자가 특정 자원·기능에 접근할 권한이 있는지 판단하는 것이다. 순서상 인증이 먼저다. SI 시스템의 역할 기반 접근제어(관리자/일반)는 이 둘을 명확히 나눠 설계해야 한다.
Q28서술형 · REST/RESTful

REST가 무엇인지 설명하고, 어떤 API를 'RESTful하다'고 하는지 서술하시오.

모범답안 확인
모범답안
REST는 자원(Resource)을 URI로 식별하고 그 자원에 대한 행위를 HTTP 메서드(GET·POST·PUT·DELETE)로 표현하는 아키텍처 스타일이다. 자원은 명사로 URI에 담고(예: /users/1), 상태 코드로 결과를 알리며, 서버는 무상태로 동작한다. 이런 원칙(자원 중심 URI, 메서드로 행위 구분, 무상태, 표현으로 데이터 교환)을 잘 지킨 API를 RESTful하다고 한다.

채점 포인트는 자원 중심 URI, HTTP 메서드로 행위 표현, 무상태·상태코드 활용이다. 여러 외부사·모바일 앱과 인터페이스를 맞춰야 하는 SI에서 RESTful 규칙은 팀 간 API 문서·소통 비용을 크게 줄인다.
Q29서술형 · Spring IoC/DI

스프링의 IoC(제어의 역전)와 DI(의존성 주입)가 무엇인지 설명하고, 이것이 주는 이점을 서술하시오.

모범답안 확인
모범답안
IoC는 객체의 생성·생명주기 관리 제어권을 개발자가 아닌 스프링 컨테이너가 갖는 것을 말한다. DI는 그 IoC를 구현하는 방식으로, 객체가 필요로 하는 의존 객체를 컨테이너가 외부에서 주입해 준다. 덕분에 객체가 의존 대상을 직접 생성하지 않아 결합도가 낮아지고, 구현체 교체·단위 테스트(목 객체 주입)가 쉬워진다.

채점 포인트는 제어권이 컨테이너로 넘어감(IoC), 의존 객체를 외부에서 주입(DI), 결합도 저하·테스트 용이 이점이다. 모듈이 많고 담당자가 바뀌는 SI 유지보수에서 낮은 결합도는 변경 파급을 줄여 유지보수 비용을 낮춘다.
Q30서술형 · 서블릿/JSP 생명주기

서블릿의 생명주기(init/service/destroy)를 설명하고, JSP가 서블릿과 어떤 관계인지 서술하시오.

모범답안 확인
모범답안
서블릿은 최초 요청 시 init()으로 1회 초기화되고, 요청마다 service()(doGet/doPost)로 처리하며, 컨테이너 종료 시 destroy()로 자원을 해제한다. JSP는 HTML 안에 자바 코드를 섞어 쓰는 화면 기술로, 실행 시 컨테이너(WAS)가 JSP를 서블릿 자바 코드로 변환·컴파일해 실행한다. 즉 JSP도 내부적으로는 서블릿으로 동작하며, 최초 호출 때 변환 비용이 한 번 든다.

채점 포인트는 init 1회·service 매 요청·destroy 종료의 생명주기, JSP가 서블릿으로 변환된다는 관계다. 레거시 SI 시스템에 JSP/서블릿이 많아, 첫 호출 지연·공통 초기화 위치 같은 유지보수 이슈를 이해하는 데 필요하다.

이 세트 우선복습 3줄

  1. 401 vs 403 — 401은 인증(로그인) 자체가 안 됨, 403은 인증은 됐으나 권한 없음. "로그인부터 하라"와 "너는 접근 불가"를 구분해 응답해야 프론트가 로그인 유도/권한 안내를 나눌 수 있다.
  2. HTTPS 하이브리드 암호화 — 느린 비대칭키는 핸드셰이크에서 대칭키를 교환하는 데만 쓰고, 대량 본문은 빠른 대칭키로 암호화한다. "비대칭키로 본문 전체 암호화"는 함정.
  3. 필터 vs 인터셉터 — 필터는 서블릿 컨테이너 레벨(DispatcherServlet 앞뒤·인코딩/전역 로깅), 인터셉터는 스프링 MVC 내부(컨트롤러 전후·로그인/권한 체크). 동작 계층이 다르다.
