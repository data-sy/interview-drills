Q21객관식 · 트리(이진트리)

노드가 최대 2개의 자식을 갖는 트리 구조의 이름은?

  1. 그래프(Graph)
  2. 이진 트리(Binary Tree)
  3. 연결리스트(Linked List)
  4. 해시테이블(Hash Table)
정답 확인
② 이진 트리(Binary Tree)
이진 트리는 각 노드가 최대 2개(왼쪽·오른쪽)의 자식을 가지며, 탐색·정렬·인덱스 구현의 기반이 된다. 특히 균형 이진 탐색 트리는 O(log n) 탐색을 제공한다. SI에서 조직도·카테고리 계층 같은 트리형 데이터를 표현·순회할 때 쓰인다.
Q22객관식 · 정렬 복잡도 비교

다음 정렬 알고리즘 중 평균 시간복잡도가 O(n log n)이 아닌 것은?

  1. 병합 정렬(Merge Sort)
  2. 퀵 정렬(Quick Sort)
  3. 힙 정렬(Heap Sort)
  4. 삽입 정렬(Insertion Sort)
정답 확인
④ 삽입 정렬(Insertion Sort)
삽입 정렬은 평균/최악 O(n²)이고(거의 정렬된 데이터엔 빠름), 병합·퀵·힙 정렬은 평균 O(n log n)이다. SI에서 대량 데이터를 애플리케이션에서 직접 정렬해야 한다면 O(n log n) 계열을 쓰거나 DB 정렬에 위임하는 것이 성능상 유리하다.
Q23객관식 · 메모리(스택/힙)

프로그램 실행 시 메모리 영역에 대한 설명으로 옳은 것은?

  1. 지역 변수와 함수 호출 정보는 힙(Heap) 영역에 저장된다
  2. 동적 할당된 객체는 스택(Stack) 영역에 저장된다
  3. 스택은 함수 호출과 함께 자동으로 할당·해제된다
  4. 힙 영역은 스택보다 항상 접근 속도가 빠르다
정답 확인
③ 스택은 함수 호출과 함께 자동으로 할당·해제된다
스택은 지역 변수·호출 프레임을 담고 함수 진입/종료에 맞춰 자동 관리되며, 동적 할당 객체는 힙에 저장되어 GC나 명시적 해제로 정리된다. SI 자바 애플리케이션에서 메모리 누수는 대개 힙에 참조가 남아 GC가 못 지우는 상황이라 힙/스택 구분이 진단의 기초다.
Q24객관식 · TCP/IP 4계층

TCP/IP 4계층 모델에서 HTTP, FTP, DNS 같은 프로토콜이 속하는 계층은?

  1. 네트워크 인터페이스 계층
  2. 인터넷 계층
  3. 전송 계층
  4. 응용(Application) 계층
정답 확인
④ 응용(Application) 계층
TCP/IP 4계층(응용·전송·인터넷·네트워크 인터페이스)에서 HTTP/FTP/DNS 같은 응용 서비스 프로토콜은 최상위 응용 계층에 속한다. TCP/UDP는 전송, IP는 인터넷 계층이다. 고객사 연계 프로토콜을 설계할 때 각 계층의 역할 구분이 통신 규격 협의의 기준이 된다.
Q25객관식 · 인덱스 B-tree

관계형 DB 인덱스가 주로 B-tree(B+tree) 구조를 사용하는 이유로 옳은 것은?

  1. 데이터를 정렬 없이 무작위로 저장하기 위해
  2. 디스크 용량을 전혀 사용하지 않기 위해
  3. 범위 검색과 정렬된 탐색을 O(log n)으로 효율화하기 위해
  4. 삽입·삭제 시 재구성이 전혀 필요 없기 때문에
정답 확인
③ 범위 검색과 정렬된 탐색을 O(log n)으로 효율화하기 위해
B-tree 계열은 균형 트리라 탐색·삽입·삭제가 O(log n)이고, 정렬된 구조라 범위 검색(BETWEEN, 부등호)에 강하다. SI에서 조회 조건 컬럼에 인덱스를 걸어 대량 테이블의 WHERE·ORDER BY 성능을 확보하는 것이 튜닝의 핵심이다.
Q26객관식 · 해시테이블

해시테이블에서 서로 다른 키가 같은 해시값(버킷)을 갖는 현상과 그 해결책을 옳게 짝지은 것은?

  1. 오버플로 - 정규화
  2. 충돌(Collision) - 체이닝/개방 주소법
  3. 언더플로 - 인덱싱
  4. 해시 미스 - 정렬
정답 확인
② 충돌(Collision) - 체이닝/개방 주소법
서로 다른 키가 같은 버킷에 매핑되는 것을 충돌이라 하며, 연결리스트로 잇는 체이닝이나 다른 빈 칸을 찾는 개방 주소법으로 해결한다. 충돌이 적으면 평균 O(1) 검색이 가능하다. SI에서 세션·캐시 저장소 키 설계 시 충돌을 줄이는 해시 분산이 성능에 영향을 준다.
Q27객관식 · 비트연산

정수 연산에서 5 & 3(비트 AND)의 결과는? (5=101₂, 3=011₂)

  1. 1
  2. 3
  3. 7
  4. 8
정답 확인
① 1
비트 AND는 두 비트가 모두 1일 때만 1이므로 101 & 011 = 001 = 1이다. AND/OR/XOR/시프트는 플래그·마스킹에 쓰인다. SI에서 사용자 권한을 비트 플래그로 묶어 관리하고 권한 & 특정권한으로 보유 여부를 빠르게 검사하는 패턴이 자주 등장한다.
Q28객관식 · GC(가비지컬렉션)

자바 등에서 가비지 컬렉션(Garbage Collection)에 대한 설명으로 옳은 것은?

  1. 더 이상 참조되지 않는 힙 객체의 메모리를 자동으로 회수한다
  2. 스택 영역의 지역 변수를 개발자가 수동으로 해제하는 기법이다
  3. GC가 동작하는 동안 성능 영향이 전혀 없다
  4. 참조가 남아 있어도 오래된 객체는 무조건 회수한다
정답 확인
① 더 이상 참조되지 않는 힙 객체의 메모리를 자동으로 회수한다
GC는 도달 불가능(참조 끊긴) 힙 객체를 찾아 자동 회수해 메모리 누수를 줄인다. 다만 GC 수행 중 일시 정지(stop-the-world) 등 성능 영향이 있고, 참조가 남으면 회수되지 않는다. SI 장기 구동 서버에서 불필요한 참조를 안 끊어 힙이 차오르는 누수가 대표 장애 원인이다.
Q29단답 · 덱(Deque)

양쪽 끝(앞·뒤) 모두에서 삽입과 삭제가 가능한 자료구조의 이름을 쓰시오.

모범답안 확인
모범답안 · 덱(Deque, Double-Ended Queue)
덱은 큐와 스택의 동작을 모두 포함해 양쪽에서 넣고 뺄 수 있는 자료구조다. 앞뒤 삽입/삭제가 자유로워 슬라이딩 윈도우, 최근 항목 관리 등에 쓰인다. SI에서 최근 조회 목록처럼 앞은 최신 추가·뒤는 오래된 것 제거가 필요한 캐시 구현에 유용하다.
Q30서술형 · DNS

DNS(Domain Name System)의 역할을 설명하고, 사용자가 브라우저에 도메인을 입력했을 때 DNS가 어떤 변환을 수행하는지 서술하시오.

모범답안 확인
모범답안
DNS는 사람이 읽기 쉬운 도메인 이름(예: www.example.com)을 컴퓨터가 통신에 사용하는 IP 주소(예: 93.184.216.34)로 변환해주는 시스템이다. 사용자가 도메인을 입력하면 브라우저는 DNS 서버(로컬 캐시→리졸버→루트/TLD/권한 서버 순)에 질의해 해당 도메인의 IP를 받아온 뒤, 그 IP로 실제 서버에 접속한다.

채점 포인트는 "도메인↔IP 변환"과 계층적 질의 흐름이다. SI에서 고객사 서버 이전·도메인 변경 시 DNS 레코드(A 레코드) 갱신과 TTL로 인한 전파 지연을 이해해야 배포·장애 대응이 매끄럽다.

이 세트 우선복습 3줄

  1. 정렬 복잡도 — 병합·퀵·힙은 평균 O(n log n), 삽입 정렬은 O(n²). 대량 데이터는 O(n log n) 계열이나 DB 정렬에 위임.
  2. 스택 vs 힙 — 지역 변수·호출 프레임은 스택(자동 할당·해제), 동적 할당 객체는 힙(GC/해제 대상). 메모리 누수 진단의 기초.
  3. 해시 충돌 — 서로 다른 키가 같은 버킷에 매핑되는 현상. 체이닝/개방 주소법으로 해결하며 충돌이 적으면 평균 O(1).
