트리 순회·정렬·동기화부터 계층 매핑·격리 수준까지

Q41객관식 · 트리 순회

아래 이진 트리를 중위 순회(In-order)한 결과는?

        A
       / \
      B   C
     / \
    D   E
  1. A B D E C
  2. D B E A C
  3. D E B C A
  4. A B C D E
정답 확인
② D B E A C
중위 순회는 왼쪽 → 루트 → 오른쪽 순서라 D B E A C가 된다(전위는 루트 먼저, 후위는 루트 나중). 이진 탐색 트리를 중위 순회하면 오름차순 정렬 결과가 나온다. SI에서 카테고리·조직 트리를 정해진 순서로 출력·집계할 때 순회 방식 선택이 결과 순서를 좌우한다.
Q42객관식 · 퀵정렬

퀵 정렬(Quick Sort)에 대한 설명으로 옳은 것은?

  1. 평균 O(n log n)이지만 피벗 선택이 나쁘면 최악 O(n²)이 될 수 있다
  2. 항상 O(n log n)을 보장하는 안정 정렬이다
  3. 추가 메모리를 O(n)만큼 반드시 사용한다
  4. 데이터가 이미 정렬되어 있을 때 항상 가장 빠르다
정답 확인
① 평균 O(n log n)이지만 피벗 선택이 나쁘면 최악 O(n²)이 될 수 있다
퀵 정렬은 피벗 기준 분할·정복으로 평균 O(n log n)이지만, 피벗이 계속 한쪽으로 치우치면(정렬된 데이터에 첫 원소 피벗) 최악 O(n²)이 되고 불안정 정렬이다. SI에서 라이브러리 정렬을 쓸 때 최악 케이스와 안정성 요구를 알아야 알고리즘을 올바르게 선택한다.
Q43객관식 · 교착상태 회피

교착상태를 "회피(avoidance)"하기 위해 안전 상태를 유지하며 자원을 할당하는 대표 알고리즘은?

  1. 다익스트라 최단경로
  2. 라운드 로빈
  3. 은행원 알고리즘(Banker's Algorithm)
  4. 퀵 정렬
정답 확인
③ 은행원 알고리즘(Banker's Algorithm)
은행원 알고리즘은 자원 할당 요청을 승인하기 전 시스템이 안전 상태를 유지하는지 검사해 교착상태를 회피한다. SI 시스템에서 한정된 커넥션 풀·락 자원을 여러 트랜잭션이 요청할 때, 자원 할당 순서·상한을 관리해 교착을 예방하는 사고방식과 맞닿아 있다.
Q44객관식 · OSI↔TCP/IP 매핑

OSI 7계층과 TCP/IP 4계층 대응으로 옳은 것은?

  1. OSI 전송 계층 ↔ TCP/IP 인터넷 계층
  2. OSI 물리 계층 ↔ TCP/IP 전송 계층
  3. OSI 네트워크 계층 ↔ TCP/IP 응용 계층
  4. OSI 응용·표현·세션 계층 ↔ TCP/IP 응용 계층
정답 확인
④ OSI 응용·표현·세션 계층 ↔ TCP/IP 응용 계층
TCP/IP 응용 계층은 OSI의 응용·표현·세션 3개를 묶고, 전송↔전송, 네트워크↔인터넷, 물리·데이터링크↔네트워크 인터페이스로 대응된다. SI에서 통신 규격/패킷 분석 도구를 이해할 때 두 모델의 대응 관계가 문제 계층 진단의 공통 언어가 된다.
Q45객관식 · ACID 격리성

트랜잭션 격리 수준이 낮을 때 발생할 수 있는 현상이 아닌 것은?

  1. Dirty Read(커밋되지 않은 데이터를 읽음)
  2. Non-Repeatable Read(같은 조회가 다른 값 반환)
  3. Phantom Read(재조회 시 새로운 행이 나타남)
  4. Deadlock(두 트랜잭션이 서로의 자원을 대기)
정답 확인
④ Deadlock(두 트랜잭션이 서로의 자원을 대기)
격리 수준이 낮을수록 Dirty/Non-Repeatable/Phantom Read가 생길 수 있으나, 교착상태(Deadlock)는 자원 잠금 대기 문제로 격리 수준 저하로 생기는 읽기 이상현상과는 범주가 다르다. SI에서 정합성이 중요한 집계·정산은 격리 수준을 높이고, 조회 위주는 낮춰 성능과 정합성을 절충한다.
Q46객관식 · 그래프 탐색

그래프 탐색에서 시작 정점과 가까운 정점부터 레벨 순으로 방문하며 주로 큐를 사용하는 방식은?

  1. 너비 우선 탐색(BFS)
  2. 깊이 우선 탐색(DFS)
  3. 이진 탐색(Binary Search)
  4. 힙 정렬(Heap Sort)
정답 확인
① 너비 우선 탐색(BFS)
BFS는 큐를 이용해 가까운 정점부터 층별로 방문해 최단 경로(간선 수 기준) 탐색에 적합하고, DFS는 스택/재귀로 한 방향을 끝까지 파고든다. SI에서 조직 관계·연계 흐름에서 "가장 짧은 연결 단계"를 찾을 때 BFS를 활용한다.
Q47객관식 · 진법(2진↔16진)

2진수 11111111(2)을 16진수로 변환한 값은?

  1. 0xEF
  2. 0xFF
  3. 0xF0
  4. 0x1F
정답 확인
② 0xFF
4비트씩 끊으면 1111(=F) 1111(=F)이라 0xFF이며 10진수로는 255다(1바이트 최댓값). SI에서 색상값(#FFFFFF), 인코딩, 바이트 단위 데이터를 다룰 때 2진↔16진 변환 감각이 값 해석의 기본이 된다.
Q48객관식 · 뮤텍스 vs 세마포어

동기화 기법인 뮤텍스(Mutex)와 세마포어(Semaphore)의 차이로 옳은 것은?

  1. 뮤텍스는 동시에 여러 스레드가 임계구역에 들어가도록 허용한다
  2. 세마포어는 항상 0과 1 두 값만 가지는 잠금 장치다
  3. 세마포어는 카운터로 동시 접근 가능한 자원 수를 제어하고, 뮤텍스는 한 번에 하나의 스레드만 소유하는 잠금이다
  4. 뮤텍스는 값이 여러 개인 계수형 동기화 도구다
정답 확인
③ 세마포어는 카운터로 동시 접근 가능한 자원 수를 제어하고, 뮤텍스는 한 번에 하나의 스레드만 소유하는 잠금이다
뮤텍스는 상호배제를 위한 잠금으로 한 번에 한 스레드만 임계구역을 소유(소유권 개념)하고, 세마포어는 카운터(counting)로 동시 접근 가능한 자원 개수를 제어한다(이진 세마포어만 0/1). SI 다중 스레드 배치나 커넥션 풀에서 동시 접근 수를 제한할 땐 세마포어, 단일 공유 자원 보호엔 뮤텍스를 써서 경합·데이터 깨짐 같은 동시성 장애를 예방한다.
Q49단답 · 해시테이블

충돌이 적은 이상적인 상황에서 해시테이블의 검색·삽입·삭제 평균 시간복잡도를 Big-O로 쓰시오.

모범답안 확인
모범답안 · O(1)
좋은 해시 함수로 충돌이 거의 없으면 키를 해시값으로 곧장 위치를 찾으므로 평균 O(1)이다(최악, 충돌 몰림 시엔 O(n)). SI에서 세션·캐시·코드 매핑처럼 키로 즉시 값을 찾아야 하는 저장소에 해시 기반 구조(HashMap 등)를 쓰는 이유다.
Q50서술형 · 프로세스 상태전이

프로세스의 주요 상태(생성/준비/실행/대기/종료) 중 준비(Ready)·실행(Running)·대기(Blocked/Waiting) 세 상태 사이의 전이를 설명하시오.

모범답안 확인
모범답안
준비(Ready)는 CPU 할당만 기다리는 상태, 실행(Running)은 CPU를 받아 명령을 수행 중인 상태, 대기(Blocked)는 I/O 등 이벤트 완료를 기다리는 상태다. 전이는 ① 준비→실행: 스케줄러가 CPU를 배정(dispatch), ② 실행→준비: 타임 슬라이스 만료 등으로 선점(preempt), ③ 실행→대기: I/O 요청으로 블록, ④ 대기→준비: I/O 완료로 다시 준비 큐로 이동한다.

채점 포인트는 세 상태의 의미와 네 방향 전이(디스패치·선점·블록·완료)다. SI 서버에서 스레드가 DB/파일 I/O로 대기 상태에 오래 머물면 처리량이 떨어지므로, 비동기·논블로킹 처리로 대기 시간을 줄이는 설계와 직결된다.

이 세트 우선복습 3줄

  1. 트리 순회 순서 — 중위(In-order)는 왼쪽 → 루트 → 오른쪽. 이진 탐색 트리를 중위 순회하면 오름차순이 나온다. 전위=루트 먼저, 후위=루트 나중으로 구분.
  2. 뮤텍스 vs 세마포어 — 뮤텍스는 한 번에 한 스레드만 소유하는 잠금(상호배제), 세마포어는 카운터로 동시 접근 자원 수를 제어. 이진 세마포어만 0/1.
  3. 격리 수준 ≠ 교착상태 — 격리 수준 저하로 생기는 건 Dirty·Non-Repeatable·Phantom Read. Deadlock은 자원 잠금 대기 문제로 범주가 다르다.
