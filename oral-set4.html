오버라이딩·접근제어자부터 데이터 이관·갈등 대응까지

각 질문을 소리 내어 말하듯 직접 답해 보고, 모범답변 확인을 눌러 요지와 비교한 뒤 스스로 채점하세요.

Q31구두 · 설명형 · 오버라이딩을 왜 쓰나

메서드 오버라이딩을 왜 사용하는지, 어떤 상황에서 쓰는지 설명해보세요.

모범답변 요지
  • 부모/인터페이스가 정한 메서드를 자식이 목적에 맞게 재정의
  • 다형성과 결합해 같은 호출로 객체마다 다른 동작을 수행
  • 공통 골격은 상위에 두고 세부 동작만 하위에서 바꿈(템플릿 확장)
  • 실무 연결: 공통 처리 흐름은 두고 고객사·업무별 세부 규칙만 재정의하면, 기존 코드를 건드리지 않고 확장할 수 있어 유지보수에 유리함을 언급
Q32구두 · 설명형 · 접근제어자

자바의 접근제어자 4가지(public/protected/default/private)의 범위를 설명해보세요.

모범답변 요지
  • private: 같은 클래스 내에서만 접근
  • default(package-private): 같은 패키지 내 접근
  • protected: 같은 패키지 + 다른 패키지의 자식 클래스
  • public: 어디서나 접근
  • 실무 연결: 필요 이상으로 public을 남발하면 내부 구현이 외부에 종속돼 변경이 어려워지므로, 최소 공개 원칙이 SI 코드 유지보수성을 높인다는 점 강조
Q33구두 · SI 실무형 · 문서화

개발 과정에서 문서를 어느 정도로, 왜 남기는 게 좋다고 생각하나요?

모범답변 요지
  • 요구사항·설계·API·변경 이력 등 핵심은 문서로 남김
  • 미래의 나와 인수인계 받을 동료를 위한 것(속도·오해 방지)
  • 코드로 드러나는 것은 중복 문서화 지양, 의도·배경 위주로 기록
  • 실무 연결: 담당자 교체가 잦은 SI 특성상 문서가 곧 인수인계 자산이라, "문서 없는 시스템은 위험 자산"이라는 관점을 강조
Q34구두 · 설명형 · 서브쿼리

서브쿼리가 무엇이고 어떤 종류가 있는지 설명해보세요.

모범답변 요지
  • 쿼리 안에 포함된 또 다른 SELECT문
  • 스칼라 서브쿼리(단일 값), 인라인 뷰(FROM절 가상 테이블), 상관 서브쿼리(바깥 행 참조)
  • WHERE·SELECT·FROM 등 위치에 따라 용도가 다름
  • 실무 연결: 통계·집계 리포트에서 "평균보다 높은 건" 같은 조건을 서브쿼리로 자주 표현하며, 상관 서브쿼리는 성능에 주의해야 함을 언급
Q35구두 · 설명형 · HTTP 상태코드

HTTP 상태코드의 2xx, 3xx, 4xx, 5xx가 각각 무엇을 뜻하는지 설명해보세요.

모범답변 요지
  • 2xx: 성공(200 OK, 201 Created)
  • 3xx: 리다이렉션(301 영구 이동, 302 임시)
  • 4xx: 클라이언트 오류(400 잘못된 요청, 401 인증, 403 권한, 404 없음)
  • 5xx: 서버 오류(500 내부 오류, 503 이용 불가)
  • 실무 연결: 장애 신고 시 상태코드로 문제가 클라이언트인지 서버인지 1차 구분되므로, 코드 해석이 빠른 원인 분류의 출발점임을 강조
Q36구두 · SI 실무형 · 신기술 학습법

프로젝트에서 처음 보는 기술을 써야 할 때 어떻게 학습하나요?

모범답변 요지
  • 공식 문서·핵심 개념부터 파악하고 작은 예제로 직접 실행
  • 프로젝트에 필요한 범위부터 우선 습득(전부가 아니라 목표 지향)
  • 막히면 검색·동료·커뮤니티 활용, 배운 것을 정리·공유
  • 실무 연결: 고객사마다 기술 스택이 다른 SI 특성상 "빠르게 배워 적용하는 능력"이 핵심 역량이며, 스스로 학습하는 태도를 강조
Q37구두 · 설명형 · Big-O

시간복잡도(Big-O)가 무엇이고 왜 신경 써야 하는지 설명해보세요.

모범답변 요지
  • 입력 크기 n이 커질 때 연산량이 늘어나는 정도를 표기(O(1), O(log n), O(n), O(n²) 등)
  • 최악의 경우 성능 경향을 비교하는 척도
  • 데이터가 커지면 비효율 알고리즘(예: 이중 반복 O(n²))은 급격히 느려짐
  • 실무 연결: 개발 땐 데이터가 적어 안 느껴지다 운영에서 데이터가 쌓이면 성능 문제가 터지므로, 데이터 규모를 가정한 복잡도 감각이 필요함을 강조
Q38구두 · 설명형 · String vs StringBuilder

String과 StringBuilder의 차이를 설명하고 언제 무엇을 쓰는지 말해보세요.

모범답변 요지
  • String은 불변(immutable)이라 문자열을 더할 때마다 새 객체 생성
  • StringBuilder는 가변이라 내부 버퍼에서 수정, 반복 문자열 조합에 효율적
  • 단순·소량이면 String, 반복문에서 문자열을 이어 붙이면 StringBuilder
  • 실무 연결: 대량 데이터로 SQL·리포트 문자열을 반복 생성하는 배치에서 String 연결은 성능·메모리 낭비가 커, 실무 성능 이슈로 직결됨을 언급
Q39구두 · SI 실무형 · 데이터 마이그레이션

기존 시스템의 데이터를 새 시스템으로 옮기는 작업을 맡았다면 어떻게 진행하겠습니까?

모범답변 요지
  • 원본·대상 스키마 매핑과 데이터 정합성 규칙을 먼저 정의
  • 소량으로 시범 이관 후 검증(건수·합계·샘플 비교)
  • 실패·중단 대비 롤백·재실행 방안 마련, 백업 필수
  • 실제 이관은 서비스 영향 적은 시간에, 이관 후 검증까지
  • 실무 연결: 데이터 이관은 SI에서 사고 시 복구가 어려운 고위험 작업이라, 백업·검증·단계적 진행이라는 안전 절차를 강조
Q40구두 · 인성/동기형 · 갈등 경험

팀 작업에서 의견 충돌이 있었던 경험과, 어떻게 풀었는지 말해보세요.

모범답변 요지
  • 구체적 상황과 상대 입장을 객관적으로 서술
  • 감정이 아니라 근거·데이터·목표를 기준으로 대화한 과정
  • 서로 절충하거나 합리적 기준으로 결론 낸 결과와 배운 점
  • 실무 연결: SI는 다양한 이해관계자와 부딪히므로, 갈등을 회피하지 않되 감정이 아닌 문제 중심으로 푸는 태도를 보여주는 것이 핵심

이 세트 우선복습 3줄

  1. 접근제어자 범위 — public > protected > default > private. protected는 "같은 패키지 + 다른 패키지의 자식"까지라는 점이 자주 헷갈리는 포인트.
  2. HTTP 상태코드 4xx vs 5xx — 4xx는 클라이언트 잘못(400/401/403/404), 5xx는 서버 잘못(500/503). 장애 1차 분류의 기준선.
  3. String vs StringBuilder — String은 불변이라 반복 연결 시 매번 새 객체 생성. 반복문 문자열 조합은 StringBuilder가 성능·메모리에 유리.
